# 🚀 Шпаргалка команд проекта ActionMachine

> Полный справочник команд для разработки, тестирования и деплоя проекта ActionMachine
>
> **Основан на современном стеке:** `uv` + `taskipy` + `docker`

---

## 📦 **UV — Менеджер пакетов нового поколения**

> `uv` — это ультра-быстрый менеджер пакетов, написанный на Rust. Заменяет `pip`, `pip-tools` и `virtualenv`.

```bash
# ------------------------------------------------------------
# Установка UV (если ещё не установлен)
# ------------------------------------------------------------
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.zshrc  # или ~/.bashrc

# ------------------------------------------------------------
# Работа с зависимостями
# ------------------------------------------------------------
# Установка всех зависимостей (все группы: dev, test, docs)
uv sync --all-groups

# Установка только production зависимостей (для деплоя)
uv sync --no-dev

# Установка только для тестов (CI/CD)
uv sync --group test

# Установка только для разработки
uv sync --group dev

# Добавить новую зависимость в проект
uv add package-name

# Добавить dev-зависимость
uv add --dev package-name

# Обновить все зависимости до последних версий
uv sync --upgrade

# Показать установленные пакеты
uv pip list

# Показать дерево зависимостей
uv pip tree

# ------------------------------------------------------------
# Виртуальное окружение
# ------------------------------------------------------------
# Создать новое виртуальное окружение
uv venv

# Активировать (после создания)
source .venv/bin/activate

# Деактивировать
deactivate

# Удалить окружение (если нужно пересоздать)
rm -rf .venv
```

## 🎯 **Taskipy — Система автоматизации задач**

> Заменяет `run_checks.sh`. Все задачи описаны в `pyproject.toml` и запускаются одной командой.

```bash
# ------------------------------------------------------------
# Справка
# ------------------------------------------------------------
# Показать все доступные задачи
uv run task --list

# ------------------------------------------------------------
# 🔍 ПРОВЕРКИ КАЧЕСТВА КОДА
# ------------------------------------------------------------

# Линтер (ruff) — проверка стиля, поиск ошибок
uv run task lint

# Авто-исправление всех проблем, которые можно исправить
uv run task lint-fix

# Проверка типов (mypy) — строгая статическая типизация
uv run task typecheck

# Полный линтер (pylint) — глубокий анализ кода
uv run task pylint

# Поиск мёртвого кода (vulture) — неиспользуемые функции/переменные
uv run task dead

# ------------------------------------------------------------
# ✨ ФОРМАТИРОВАНИЕ КОДА
# ------------------------------------------------------------

# Отформатировать код (black) — беспощадный форматтер
uv run task format

# Проверить форматирование (без изменений)
uv run task format-check

# Отсортировать импорты (isort) — стандартизация порядка импортов
uv run task sort

# Проверить сортировку импортов
uv run task sort-check

# ------------------------------------------------------------
# 🧪 ТЕСТИРОВАНИЕ
# ------------------------------------------------------------

# Запустить все тесты
uv run task test

# Запустить тесты с измерением покрытия кода
uv run task test-cov
# После выполнения: открыть htmlcov/index.html в браузере

# Запустить только быстрые тесты (без маркера @pytest.mark.slow)
uv run task test-fast

# Запустить только unit-тесты
uv run task test-unit

# Запустить только интеграционные тесты
uv run task test-int

# ------------------------------------------------------------
# 📊 МЕТРИКИ КОДА
# ------------------------------------------------------------

# Цикломатическая сложность (оценка A, B, C...)
# A = отлично, F = очень плохо
uv run task cc

# Индекс поддерживаемости (maintainability index)
# Насколько легко поддерживать код
uv run task mi

# Сырые метрики (количество строк, комментариев, пустых строк)
uv run task raw

# ------------------------------------------------------------
# 🏗️ АРХИТЕКТУРНЫЕ ПРОВЕРКИ
# ------------------------------------------------------------

# Проверка зависимостей между модулями (import-linter)
# Например: Core не должен импортировать Auth
uv run task imports

# ------------------------------------------------------------
# 🚀 КОМБО-ЗАДАЧИ (цепочки команд)
# ------------------------------------------------------------

# ПОЛНАЯ ПРОВЕРКА — аналог старого run_checks.sh
# Запускает: линтер + типы + pylint + мёртвый код + тесты + метрики
uv run task check-all

# БЫСТРАЯ ПРОВЕРКА ПЕРЕД КОММИТОМ
# Запускает: формат + сортировка + авто-исправление + типы + быстрые тесты
uv run task pre-commit
```

## 🐳 **Docker — Контейнеризация**

> Команды для работы с Docker-образами и контейнерами проекта

```bash
# ------------------------------------------------------------
# Работа с docker-compose (локальная разработка)
# ------------------------------------------------------------
# Перейти в папку с docker-compose (если она отдельная)
cd ~/kanban_assistant_docker

# Перезапуск всех контейнеров (остановить и запустить заново)
docker-compose down
docker-compose up -d

# Принудительная перекачка последнего образа
docker-compose pull python

# Пересоздать контейнер с новым образом (без остановки других)
docker-compose up -d --force-recreate python

# Пересобрать образ с актуальными файлами (если меняли Dockerfile)
docker-compose build python

# Запустить все контейнеры в фоне
docker-compose up -d

# ------------------------------------------------------------
# Сборка и публикация образа в GitVerse registry
# ------------------------------------------------------------
# Перейти в корень проекта
cd ~/PythonDev/kanban_assistant

# Собрать образ вручную
docker build -t gitverse.ru/bystrov.maxim/kanban_assistant:latest .

# Запушить в registry (требуется авторизация)
docker push gitverse.ru/bystrov.maxim/kanban_assistant:latest

# ------------------------------------------------------------
# Работа с запущенным контейнером
# ------------------------------------------------------------
# Проверить наличие файлов в контейнере
docker-compose exec python ls -la /app/Utils

# Запустить тестовые скрипты внутри контейнера
docker-compose exec python python /app/Utils/test_init_db.py
docker-compose exec python python /app/Utils/test_youtrack_fetch_issue_to.py
docker-compose exec python python /app/Utils/test_youtrack_issue_to_csv.py
docker-compose exec python python /app/Utils/test_youtrack_issue_to_postgres.py

# Запустить только контейнер youtrack (без основного приложения)
docker-compose up -d youtrack

# Посмотреть логи контейнера
docker-compose logs -f python

# Зайти в контейнер в интерактивном режиме
docker-compose exec python bash
```

## 📝 **Git — Система контроля версий**

```bash
# Переключиться на другую ветку

# Создать тег для релиза
git tag v1.0.0

# Отправить теги в remote
git push --tags
```

## 🧪 **Статические анализаторы (прямой вызов)**

> Эти команды теперь заменены на `task`, но оставлены для истории и отладки

```bash
# ------------------------------------------------------------
# Mypy — строгая проверка типов
# ------------------------------------------------------------
mypy --strict \
     --follow-imports=normal \
     --ignore-missing-imports \
     --warn-unreachable \
     --no-implicit-reexport \
     src/action_machine/

# ------------------------------------------------------------
# Radon — метрики кода
# ------------------------------------------------------------
# Цикломатическая сложность (-s = буквенная оценка A, B, C...)
radon cc src/action_machine/ -s

# Индекс поддерживаемости (mi)
radon mi src/action_machine/ -s

# Сырые метрики (строки, комментарии)
radon raw src/action_machine/ -s

# ------------------------------------------------------------
# Vulture — поиск мёртвого кода
# ------------------------------------------------------------
# whitelist теперь в pyproject.toml, можно запускать без файла
vulture src/action_machine/
```

## 🚀 **Быстрый старт после клонирования проекта**

> Что делать, если ты скачал проект с git в первый раз

```bash
# 1. Клонировать репозиторий
git clone https://gitverse.ru/bystrov.maxim/kanban_assistant.git
cd kanban_assistant

# 2. Установить uv (если не установлен)
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.zshrc

# 3. Создать виртуальное окружение
uv venv
source .venv/bin/activate

# 4. Установить все зависимости (включая dev, test, docs)
uv sync --all-groups

# 5. Проверить что всё работает
uv run task test           # тесты должны пройти
uv run task pre-commit     # быстрая проверка кода
uv run task check-all      # полная проверка

# 6. Создать .env файл для локальных переменных (если нужно)
cp .env.example .env
# Отредактировать .env под свои нужды
```

## 🆘 **Полезные утилиты и команды**

> Вспомогательные команды для повседневной работы

```bash
# ------------------------------------------------------------
# Tree — просмотр структуры проекта
# ------------------------------------------------------------
# Установить tree (если не установлен)
brew install tree              # на macOS
# или
apt-get install tree           # на Linux

# Посмотреть структуру проекта (2 уровня, исключая мусор)
tree -L 2 -I '__pycache__|*.pyc|.venv|archive|build|dist'

# ------------------------------------------------------------
# Права на выполнение
# ------------------------------------------------------------
# Сделать скрипт исполняемым
chmod +x scripts/deploy.sh

# ------------------------------------------------------------
# Поиск в файлах
# ------------------------------------------------------------
# Найти все вхождения слова в .py файлах
grep -r "TODO" src/action_machine/

# Найти файлы по имени
find . -name "*test*.py"

# ------------------------------------------------------------
# Очистка проекта
# ------------------------------------------------------------
# Удалить все кэши Python
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -type f -name "*.pyc" -delete

# Удалить папку сборки
rm -rf build/ dist/ *.egg-info/
```
