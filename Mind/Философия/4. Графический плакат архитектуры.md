# 🏛 Графический плакат архитектуры ActionEngine (AOA)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                             Action‑Oriented Architecture                      │
│                                    (AOA)                                      │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│                                Транспортные слои                              │
│                     (HTTP/FastAPI, MCP, CLI, Background Jobs)                │
│                                                                              │
│  - Принимают внешний запрос                                                  │
│  - Создают Context (UserInfo, RequestInfo, EnvironmentInfo) [1]              │
│  - Передают Params → в ActionMachine                                         │
│  - Не содержат бизнес‑логики                                                 │
│  - Не создают зависимости                                                    │
└──────────────────────────────────────────────────────────────────────────────┘

                                     │
                                     ▼

┌──────────────────────────────────────────────────────────────────────────────┐
│                                 ActionProductMachine                         │
│                     Ядро исполнения (качество: A по MI) [1]                  │
│                                                                              │
│  1. ↑ nest_level                                                             │
│  2. Проверка ролей CheckRoles (все проверки: A) [1]                          │
│  3. Генерация события global_start → плагины                                 │
│  4. Выполнение аспектов:                                                     │
│        before:aspect → aspect() → after:aspect                               │
│     (порядок определяется по co_firstlineno) [1]                             │
│  5. Выполнение summary‑аспекта                                               │
│  6. global_finish → плагины                                                  │
│  7. ↓ nest_level                                                             │
│                                                                              │
│  Кэширует:                                                                   │
│     - аспекты (_aspect_cache)                                                │
│     - фабрики зависимостей (_factory_cache)                                  │
│     - обработчики плагинов (_plugin_cache)                                   │
└──────────────────────────────────────────────────────────────────────────────┘

                                     │
                   +-----------------┼-----------------+
                   │                                   │
                   ▼                                   ▼

┌──────────────────────────────────────────────────────────────────────────────┐
│                               DependencyFactory                               │
│                               (качество: A) [1]                               │
│                                                                              │
│  - DI без магии                                                              │
│  - Зависимости → объявляются @depends                                        │
│  - Создаются лениво                      ┌─────────────────────────────┐     │
│  - Кешируются на время run()             │ deps.get(ServiceClass)       │     │
│  - Поддержка вложенных действий:         │ deps.run_action(Action, P)   │     │
│                                          └─────────────────────────────┘     │
└──────────────────────────────────────────────────────────────────────────────┘

                                     │
                                     ▼

┌──────────────────────────────────────────────────────────────────────────────┐
│                                    Actions                                   │
│                       (NotificationAction, ParentAction…) [1]                │
│                                                                              │
│  - Stateless классы                                                          │
│  - Params & Result immutable (BaseParams/BaseResult → A) [1]                 │
│  - Содержат только бизнес‑логику                                             │
│  - Не знают о Context                                                        │
│  - Получают зависимости через deps                                           │
│                                                                              │
│                   Аспекты (AspectMethod: A) [1]                             │
│                ┌───────────────────────────────────────┐                    │
│                │ @aspect("...")                         │                    │
│                │ def step(self, params, state, deps):   │                    │
│                │     return new_state                    │                    │
│                └───────────────────────────────────────┘                    │
│                ┌───────────────────────────────────────┐                    │
│                │ @summary_aspect("...")                 │                    │
│                │ def finish(self, params, state, deps): │                    │
│                │     return Result                      │                    │
│                └───────────────────────────────────────┘                    │
└──────────────────────────────────────────────────────────────────────────────┘

                                     │
                                     ▼

┌──────────────────────────────────────────────────────────────────────────────┐
│                               Plugins (Observers)                             │
│                               (качество: A) [1]                               │
│                                                                              │
│  - Подписка через @on(event_regex, class_regex)                              │
│  - Обработчики асинхронны                                                    │
│  - Ошибки могут игнорироваться (ignore_exceptions)                           │
│  - Получают Context, Params, Result, state_aspect                             │
│  - Вложенность (nest_level) → полная видимость цепочек                       │
│                                                                              │
│  События:                                                                    │
│     - global_start                                                           │
│     - before:aspect                                                          │
│     - after:aspect                                                           │
│     - global_finish                                                          │
│                                                                              │
│  Плагины НЕ могут вмешиваться в бизнес‑данные                                 │
│  (Params immutable, Result immutable, state_aspect read‑only copy)           │
└──────────────────────────────────────────────────────────────────────────────┘

                                     │
                                     ▼

┌──────────────────────────────────────────────────────────────────────────────┐
│                             Внешние интеграции                                │
│                                                                              │
│   - Логирование                                                              │
│   - Трассировка (OpenTelemetry, Jaeger)                                      │
│   - Метрики (Prometheus, Grafana)                                            │
│   - Monitoring                                                               │
│   - Аудит                                                                    │
│                                                                              │
│ Все подключаются как плагины, БЕЗ модификации бизнес‑кода                   │
└──────────────────────────────────────────────────────────────────────────────┘


################################################################################
#                 КЛЮЧЕВАЯ ИДЕЯ AOA (Action‑Oriented Architecture)             #
################################################################################

**«Бизнес‑логика живёт в Actions.  
Инфраструктура живёт в Plugins.  
Исполнение управляется Machine.  
Композиция обеспечивается DI.»**

Эта модель подтверждается качеством кода:

- ActionProductMachine → A [1]  
- DependencyFactory → A [1]  
- BaseAction/BaseParams/BaseResult → A [1]  
- String/Int/Float/Date/InstanceOf Checkers → A [1]  
- Plugin & Decorators → A [1]  
- Все тесты — A [1]  

ActionEngine формирует строгую, чистую и детерминированную архитектуру.
