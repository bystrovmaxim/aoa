# UML‑диаграмма архитектуры AOA

Этот документ описывает архитектуру ActionEngine (AOA) через UML‑диаграммы.  
Диаграмма фиксирует ключевые сущности: действия, аспекты, машину действий, ресурсные менеджеры, DI‑фабрику и плагины.  
Все связи и элементы отражены исходя из реальной структуры кода в ActionMachine, включая ActionProductMachine, DependencyFactory, BaseAction, плагины и менеджеры ресурсов.

# UML‑диаграмма основных классов (структурная)

```plantuml
@startuml
title AOA UML — Основные сущности

' ======= Абстракции =======
abstract class BaseAction {
    +get_full_class_name() : str
}
abstract class BaseParams
abstract class BaseResult
abstract class BaseActionMachine {
    +run(action, params) : Result
}
abstract class Plugin {
    +get_initial_state() : Any
    +get_handlers(event, class) : List
}

class Context {
    user : UserInfo
    request : RequestInfo
    environment : EnvironmentInfo
}

class DependencyFactory {
    +get(klass)
    +run_action(action_class, params)
}

' ======= ActionMachine =======
class ActionProductMachine {
    -_aspect_cache
    -_factory_cache
    -_plugin_cache
    -_plugin_states
    -_nest_level
    +run(action, params)
}

BaseActionMachine <|-- ActionProductMachine
ActionProductMachine --> DependencyFactory
ActionProductMachine --> Plugin
ActionProductMachine --> Context

' ======= Actions, Params, Result =======
BaseAction --> BaseParams
BaseAction --> BaseResult

class SomeAction {
    Params
    Result
}

BaseAction <|-- SomeAction

' ======= Plugins =======
class ConsoleLoggingPlugin
class AnotherLoggingPlugin
Plugin <|-- ConsoleLoggingPlugin
Plugin <|-- AnotherLoggingPlugin

' ======= Resources =======
abstract class BaseConnectionManager
abstract class BaseResourceManager

class PostgresConnectionManager
class CsvConnectionManager

BaseConnectionManager <|-- PostgresConnectionManager
BaseConnectionManager <|-- CsvConnectionManager
BaseResourceManager <|-- BaseConnectionManager

' ======= DI =======
DependencyFactory --> BaseAction

@enduml
```

# Диаграмма последовательности (Sequence Diagram) выполнения действия

Показывает жизненный цикл `machine.run(action, params)`.

```plantuml
@startuml
title Sequence: Выполнение действия в AOA

actor Caller
Caller -> ActionProductMachine : run(action, params)

ActionProductMachine -> ActionProductMachine : _check_action_roles()
ActionProductMachine -> Plugin : global_start

loop regular aspects
    ActionProductMachine -> Plugin : before:aspect
    ActionProductMachine -> Action : aspect()
    Action --> ActionProductMachine : state'
    ActionProductMachine -> Plugin : after:aspect
end

ActionProductMachine -> Action : summary_aspect()
Action --> ActionProductMachine : Result

ActionProductMachine -> Plugin : global_finish
Caller <-- ActionProductMachine : Result

@enduml
```

# Пояснения к диаграммам

## ActionMachine — сердце архитектуры
ActionProductMachine выполняет действие шаг за шагом, строго в порядке определения аспектов.  
Она отвечает за:

- вызов аспектов,
- вызов плагинов,
- ролевую модель,
- DI через DependencyFactory,
- вложенные действия (`run_action`),
- валидацию результатов через чекеры [1],
- обработку ошибок и типов [1],
- кэширование аспектов [1].

Код ActionProductMachine имеет высокий рейтинг качества (A по radon) [1].

## DependencyFactory — адаптация инфраструктуры
DependencyFactory:

- создаёт зависимости по декларациям `@depends`,
- кеширует их,
- управляет вложенными действиями.

Её методы (`get`, `run_action`) получили оценки A [1].

## Plugins — чистое наблюдение
Плагины:

- подписываются через `@on`,
- вызываются ActionMachine на этапах pipeline,
- не могут менять выполнение,
- безопасны даже при ошибках (`ignore_exceptions=True`) [1].

## Resources — адаптеры внешнего мира
Resource Managers:

- PostgresConnectionManager [1],
- CsvConnectionManager [1],
- BaseConnectionManager [1],

инкапсулируют соединения, транзакции и работу с внешними системами.

## Actions — доменная логика
Все действия stateless, используют Params/Result, аспекты и summary.  
Все аспекты и методы BaseAction, BaseParams, BaseResult имеют рейтинг A [1].
