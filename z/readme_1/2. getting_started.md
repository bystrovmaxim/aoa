# Getting Started

Добро пожаловать в ActionEngine — фреймворк для разработки бизнес‑логики нового поколения.

ActionEngine построен на принципах **Action‑Oriented Architecture (AOA)** — подхода, в котором:

- бизнес‑логика оформляется в виде *действий* (Actions)  
- каждый шаг действия оформлен как *аспект*  
- инфраструктура подключается через *плагины*  
- параметры и результаты строго типизированы  
- тестирование встроено в архитектуру по умолчанию  
- действия легко композируются и переиспользуются

# 2. Минимальный пример действия

Создадим простое действие, которое принимает параметры и возвращает результат:

```python
from dataclasses import dataclass
from ActionMachine import BaseAction, BaseParams, BaseResult
from ActionMachine import aspect, summary_aspect

class CreateOrder(BaseAction):

    @dataclass(frozen=True)
    class Params(BaseParams):
        user_id: str
        product_id: int
        quantity: int

    @dataclass(frozen=True)
    class Result(BaseResult):
        order_id: str

    @aspect("Проверка количества")
    def check_quantity(self, params, state, deps):
        if params.quantity <= 0:
            raise ValueError("Quantity must be positive")
        state["valid"] = True
        return state

    @summary_aspect("Создание заказа")
    def handle(self, params, state, deps):
        # Имитация записи заказа
        order_id = f"order-{params.user_id}-{params.product_id}"
        return CreateOrder.Result(order_id)
```

**Ключевые моменты:**

- действие — это класс  
- параметры — dataclass, наследованный от BaseParams  
- результаты — dataclass, наследованный от BaseResult  
- аспекты выполняются перед summary‑аспектом  
- summary‑аспект всегда последний и возвращает результат

# 3. Запуск действия через ActionMachine

ActionMachine управляет:

- аспектами  
- ролями  
- вызовами плагинов  
- вложенными действиями  
- валидацией  
- DI через DependencyFactory  

Минимальный запуск:

```python
from ActionMachine import ActionProductMachine, Context

context = Context()
machine = ActionProductMachine(context=context)

params = CreateOrder.Params(user_id="u1", product_id=100, quantity=2)
result = machine.run(CreateOrder(), params)

print(result.order_id)
```

# 4. Добавление зависимостей через @depends

Зависимости объявляются декларативно — никакого DI‑контейнера:

```python
from ActionMachine import depends

class EmailService:
    def send(self, to, msg):
        print(f"Email to {to}: {msg}")

@depends(EmailService, description="Почтовый сервис")
class NotifyUser(BaseAction):

    @dataclass(frozen=True)
    class Params(BaseParams):
        user: str
        msg: str

    @dataclass(frozen=True)
    class Result(BaseResult):
        ok: bool

    @summary_aspect("Отправка сообщения")
    def handle(self, params, state, deps):
        service = deps.get(EmailService)
        service.send(params.user, params.msg)
        return NotifyUser.Result(ok=True)
```

DependencyFactory создаётся автоматически — согласно radon оценён как **A** (качество кода высокое) [1].

# 5. Добавление ролей через CheckRoles

ActionEngine имеет встроенную систему ролей:

```python
from ActionMachine import CheckRoles

@CheckRoles(CheckRoles.ANY, desc="Требуется любая роль")
class SomeAction(BaseAction):
    ...
```

Все проверки проходят в `ActionProductMachine._check_action_roles`, которая имеет рейтинг **A (5)** [1].

# 6. Написание плагинов

Плагин — это **read‑only наблюдатель**.  
Он не может повлиять на бизнес‑логику, только логировать/собирать метрики.

Минимальный плагин:

```python
from ActionMachine.Plugins.Decorators import on
from ActionMachine.Plugins.Plugin import Plugin

class ConsolePlugin(Plugin):

    def get_initial_state(self):
        return {}

    @on("global_start", ".*", ignore_exceptions=True)
    async def started(self, state_plugin, event_name, action_name, params,
                      state_aspect, is_summary, deps, context, result, duration, nest_level):
        print(f"[start] {action_name} {params}")
        return state_plugin

    @on("global_finish", ".*", ignore_exceptions=True)
    async def finished(self, state_plugin, event_name, action_name, params,
                       state_aspect, is_summary, deps, context, result, duration, nest_level):
        print(f"[finish] {action_name} result={result}")
        return state_plugin
```

Модуль `Plugin` имеет рейтинг **A (59.09 MI)** [1].

Добавим плагин:

```python
machine._plugins.append(ConsolePlugin())
```

# 7. Вложенные действия

Любое действие может вызывать другое:

```python
child_result = deps.run_action(ChildAction, ChildAction.Params(5))
```

Этот механизм реализован в DependencyFactory и ActionProductMachine  
(оба имеют рейтинг A [1]).

# 8. Тестирование — встроенная фича

ActionEngine проектировался под unit‑tests *с самого начала*.

### Пример:

```python
from ActionMachine import ActionTestMachine

machine = ActionTestMachine({
    ChildAction: ChildAction.Result(20)
})

params = ParentAction.Params(7)
result = machine.run(ParentAction(), params)
assert result.result == 30
```

TestMachine:

- полностью синхронная  
- подменяет зависимости  
- принимает MockAction и side_effect  
- проверяет вызовы  
- логически изолирована  

Файл `ActionTestMachine.py` имеет рейтинг **A (72.83)** [1].
