# Финальный план рефакторинга GateCoordinator
## Гейтхосты-фасеты с транзакционным построением графа

---

## Контекст

Это неинвазивная трансформация — существующие файлы, классы и декораторы остаются на месте. Добавляются новые файлы и методы. Пользовательский код не меняется. Старый `MetadataBuilder` работает параллельно до полного перехода, после чего удаляется.

---

## 1. Архитектура

### 1.1. Центральная идея

Система строит **граф зависимостей** между объектами предметной области. Узлы графа — это классы (`Action`, `Entity`, `Role`, `Aspect`...). Рёбра — отношения между ними (`depends`, `has_aspect`, `belongs_to`...).

```
        CreateOrderAction
         /            \
    depends          has_aspect
       /                  \
 OrderEntity          ValidationAspect
       \
    belongs_to
          \
        OrderTable
```

Граф строится один раз при старте приложения. После построения — только читается.

---

### 1.2. Кто знает про классы?

Гейтхост — это маркерный миксин. Когда разработчик пишет:

```python
class CreateOrderAction(BaseAction, RoleGateHost):
    ...
```

Python автоматически регистрирует `CreateOrderAction` как наследника `RoleGateHost` через `__subclasses__()`. Никакой ручной регистрации классов не существует. Гейтхост сам знает всех своих наследников:

```
RoleGateHost.__subclasses_recursive__()
    → [CreateOrderAction, UpdateOrderAction, ...]

AspectGateHost.__subclasses_recursive__()
    → [ValidationAspect, LoggingAspect, ...]

EntityGateHost.__subclasses_recursive__()
    → [OrderEntity, PaymentEntity, ...]
```

---

### 1.3. Что делает гейтхост

Каждый гейтхост реализует три метода. Все три — `abstractmethod`. Явность обязательна:

```
┌─────────────────────────────────────────────────────────┐
│                    AbstractGateHost                      │
│                                                          │
│  inspect(cls) → FacetPayload | None      [abstract]     │
│    Подходит ли класс? Если да — вернуть payload.        │
│    Если нет — None.                                     │
│    Если класс обязан подойти, но сломан — TypeError.    │
│                                                          │
│  _build_payload(cls) → FacetPayload      [abstract]     │
│    Собрать узел и рёбра. Использует хелперы базового    │
│    класса: _make_node_name, _make_edge, _make_meta.     │
│                                                          │
│  _validate(cls, payload) → None          [abstract]     │
│    Оркестратор проверок. Вызывает отдельные методы      │
│    для каждого инварианта. Каждый инвариант —           │
│    отдельный метод, одна проверка, одно исключение.     │
│    Если проверок нет — pass явно. Это зона              │
│    ответственности каждого гейтхоста.                   │
└─────────────────────────────────────────────────────────┘
```

**Правило разделения ответственности:**
```
inspect()        → проверяет ТОЛЬКО наличие данных
_build_payload() → читает данные, формирует payload, не валидирует
_validate()      → проверяет КОРРЕКТНОСТЬ данных и инварианты
```

---

### 1.4. Хелперы базового класса

Базовый класс предоставляет хелперы, устраняющие дублирование в каждом гейтхосте:

```
┌─────────────────────────────────────────────────────────┐
│              AbstractGateHost — хелперы                  │
│                                                          │
│  _make_node_name(cls, suffix) → str                     │
│    "module.ClassName" — без префикса типа.              │
│    Ключ "тип:имя" собирает координатор.                 │
│                                                          │
│  _make_edge(target_type, target_cls,                    │
│             edge_type, is_structural,                   │
│             edge_meta) → EdgeInfo                       │
│    Собирает EdgeInfo без ручного заполнения полей.      │
│                                                          │
│  _make_meta(**kwargs) → tuple[tuple[str, Any], ...]     │
│    dict-синтаксис → иммутабельный tuple пар.            │
│                                                          │
│  __subclasses_recursive__() → list[type]                │
│    Рекурсивный обход всех наследников гейтхоста.        │
└─────────────────────────────────────────────────────────┘
```

---

### 1.5. FacetPayload — контракт между гейтхостом и координатором

```
┌─────────────────────────────────────────────────────────┐
│                     FacetPayload                        │
│                                                          │
│  node_type  : str                ← "action", "role"...  │
│  node_name  : str                ← "module.ClassName"   │
│  node_class : type               ← ссылка на класс      │
│  node_meta  : tuple[tuple]       ← специфика гейтхоста  │
│  edges      : tuple[EdgeInfo]    ← рёбра от этого узла  │
└─────────────────────────────────────────────────────────┘
```

`FacetPayload` — временный транспортный объект. Живёт только внутри `build()`. После коммита выбрасывается. В граф данные попадают как обычный `dict` — конвертация `tuple → dict` происходит один раз при коммите.

`tuple[tuple]` вместо `dict` — `frozen dataclass` должен быть хешируемым.

---

### 1.6. Два типа рёбер

```
┌─────────────────┬────────────────────────────────────────┐
│   is_structural │  Примеры        Проверка               │
├─────────────────┼────────────────────────────────────────┤
│   True          │  depends        Ацикличность обязательна│
│                 │  connection     Цикл → InvalidGraphError│
├─────────────────┼────────────────────────────────────────┤
│   False         │  has_aspect     Циклы допустимы         │
│                 │  belongs_to     Просто метаданные        │
│                 │  requires_ctx                           │
└─────────────────┴────────────────────────────────────────┘
```

---

### 1.7. GateCoordinator — сборщик графа

```python
coordinator = GateCoordinator(strict=True)\
    .register_gate_host(RoleGateHost)\
    .register_gate_host(AspectGateHost)\
    .register_gate_host(EntityGateHost)\
    .build()
```

- `build()` — ровно один раз. Повторный вызов — исключение.
- Регистрация гейтхоста после `build()` — исключение.
- Дубликат гейтхоста при регистрации — исключение.

---

### 1.8. Транзакционный build() — три фазы

Граф либо строится полностью и корректно, либо не строится вообще. Никакого частичного состояния.

```
╔══════════════════════════════════════════════════════════╗
║                    build()                               ║
╠══════════════════════════════════════════════════════════╣
║                                                          ║
║  ФАЗА 1 — СБОР (только чтение, граф не трогаем)        ║
║                                                          ║
║  для каждого gate_host:                                  ║
║    для каждого cls в __subclasses_recursive__:           ║
║      payload = gate_host.inspect(cls)                    ║
║      если payload → _validate(cls, payload)             ║
║                   → добавить в new_payloads             ║
║                                                          ║
╠══════════════════════════════════════════════════════════╣
║                                                          ║
║  ФАЗА 2 — ПРОВЕРКИ (граф не трогаем)                    ║
║                                                          ║
║  2a. Обязательные поля payload'ов не пустые             ║
║  2b. Уникальность ключей внутри new_payloads            ║
║      конфликт → DuplicateNodeError с именами            ║
║      обоих гейтхостов в сообщении                       ║
║  2c. Ссылочная целостность рёбер                        ║
║      цель ∈ new_payloads → OK                           ║
║      иначе → InvalidGraphError                          ║
║  2d. Ацикличность структурных рёбер                     ║
║      симуляция на новом пустом графе                    ║
║      цикл → InvalidGraphError                           ║
║                                                          ║
╠══════════════════════════════════════════════════════════╣
║                                                          ║
║  ФАЗА 3 — КОММИТ (только если фаза 2 прошла)           ║
║                                                          ║
║  добавить все узлы в граф                               ║
║  добавить все рёбра в граф                              ║
║  self._built = True                                      ║
║                                                          ║
╚══════════════════════════════════════════════════════════╝
```

---

### 1.9. Тестирование гейтхостов

Тестовая среда идентична продакшену. Тестовые классы с гейтхостами обязаны быть полными и валидными — они попадают в граф наравне с продакшен-классами.

Тестирование невалидных данных — изолированно через `inspect()`, без координатора:

```python
def test_role_gate_host_raises_on_empty_role_info():
    class BrokenAction(BaseAction, RoleGateHost):
        _role_info = {}

    with pytest.raises(TypeError):
        RoleGateHost.inspect(BrokenAction)
```

Маркировка тестовых узлов в графе — откладывается на этап визуализации.

## 2. Ключевые интерфейсы

### `metadata/payload.py`

```python
from __future__ import annotations
from dataclasses import dataclass
from typing import Any


@dataclass(frozen=True)
class EdgeInfo:
    target_node_type: str
    target_name: str
    edge_type: str
    is_structural: bool
    edge_meta: tuple[tuple[str, Any], ...] = ()


@dataclass(frozen=True)
class FacetPayload:
    node_type: str
    node_name: str
    node_class: type
    node_meta: tuple[tuple[str, Any], ...] = ()
    edges: tuple[EdgeInfo, ...] = ()
```

### `metadata/abstract_gate_host.py`

```python
from __future__ import annotations
from abc import ABC, abstractmethod
from typing import Any
from .payload import FacetPayload, EdgeInfo


class AbstractGateHost(ABC):

    # ── Обязательный контракт ──────────────────────────────────────────

    @classmethod
    @abstractmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        """
        None      → класс не субъект этого гейтхоста.
        payload   → класс подходит, данные собраны.
        TypeError → класс обязан подойти, но данные отсутствуют.
        """
        ...

    @classmethod
    @abstractmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        """
        Собирает FacetPayload.
        Использует хелперы: _make_node_name, _make_edge, _make_meta.
        Не валидирует данные — это задача _validate.
        """
        ...

    @classmethod
    @abstractmethod
    def _validate(cls, target_cls: type, payload: FacetPayload) -> None:
        """
        Оркестратор проверок инвариантов.
        Каждый инвариант — отдельный приватный метод.
        Если проверок нет — реализовать явно с pass.
        """
        ...

    # ── Хелперы для _build_payload ─────────────────────────────────────

    @classmethod
    def _make_node_name(cls, target_cls: type, suffix: str = "") -> str:
        """
        Возвращает "module.ClassName" — БЕЗ префикса типа.
        Ключ "тип:module.ClassName" собирает координатор.
        """
        name = f"{target_cls.__module__}.{target_cls.__qualname__}"
        return f"{name}.{suffix}" if suffix else name

    @classmethod
    def _make_edge(
        cls,
        target_node_type: str,
        target_cls: type,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """Собирает EdgeInfo без ручного заполнения каждого поля."""
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=cls._make_node_name(target_cls),
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_meta(cls, **kwargs: Any) -> tuple[tuple[str, Any], ...]:
        """dict-синтаксис → иммутабельный tuple пар для node_meta."""
        return tuple(kwargs.items())

    # ── Обход наследников ─────────────────────────────────────────────

    @classmethod
    def __subclasses_recursive__(cls) -> list[type]:
        """Рекурсивно собирает всех наследников гейтхоста."""
        result = []
        for subclass in cls.__subclasses__():
            result.append(subclass)
            result.extend(subclass.__subclasses_recursive__())
        return result
```

## 3. Требования к реализации каждого гейтхоста

### MetaGateHost

```
_build_payload:
  node_type : "meta"
  node_meta : title, description, version, tags
  edges     : ()

_validate:
  _validate_title_not_empty     — title обязан быть непустой строкой
  _validate_description_present — description обязан присутствовать
  _validate_version_format      — version соответствует semver если задан
```

### RoleGateHost

```
_build_payload:
  node_type : "role"
  node_meta : spec
  edges     : ()

_validate:
  pass  — инварианты проверены декоратором @role и в inspect()
```

### AspectGateHost

```
_build_payload:
  node_type : "aspect"
  node_meta : aspect_type ("regular" | "summary"), method_name
  edges     : ()

_validate:
  _validate_aspect_type_valid   — aspect_type ∈ {"regular", "summary"}
  _validate_method_name_present — method_name непустой
```

### CheckerGateHost

```
_build_payload:
  node_type : "checker"
  node_meta : checker_type, target_field
  edges     :
    belongs_to → аспект (is_structural=False)

_validate:
  _validate_checker_type_valid  — checker_type известный тип
  _validate_target_field_exists — target_field есть в Params/Result
```

### DependencyGateHost

```
_build_payload:
  node_type : "action"
  node_meta : ()
  edges     :
    depends → каждый класс из _depends_info (is_structural=True)

_validate:
  _validate_no_self_dependency — класс не зависит от самого себя
  _validate_deps_are_types     — каждая зависимость это type
```

### ConnectionGateHost

```
_build_payload:
  node_type : "action"
  node_meta : ()
  edges     :

_validate:
  _validate_no_self_connection — класс не соединён сам с собой
```

## 4. Примеры реализации гейтхостов

### RoleGateHost — без рёбер, простой

```python
class RoleGateHost(AbstractGateHost):

    @classmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        if not hasattr(target_cls, "_role_info"):
            return None
        if not target_cls._role_info:
            raise TypeError(
                f"{target_cls.__qualname__} наследует RoleGateHost, "
                "но _role_info отсутствует или пустой."
            )
        return cls._build_payload(target_cls)

    @classmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        return FacetPayload(
            node_type="role",
            node_name=cls._make_node_name(target_cls),
            node_class=target_cls,
            node_meta=cls._make_meta(
                spec=target_cls._role_info["spec"]
            ),
            edges=(),
        )

    @classmethod
    def _validate(
        cls, target_cls: type, payload: FacetPayload
    ) -> None:
        pass  # инварианты проверены декоратором @role и в inspect()
```

### DependencyGateHost — со структурными рёбрами

```python
class DependencyGateHost(AbstractGateHost):

    @classmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        if not hasattr(target_cls, "_depends_info"):
            return None
        if not target_cls._depends_info:
            return None
        return cls._build_payload(target_cls)

    @classmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        edges = tuple(
            cls._make_edge(
                target_node_type="action",
                target_cls=dep,
                edge_type="depends",
                is_structural=True,
            )
            for dep in target_cls._depends_info
        )
        return FacetPayload(
            node_type="action",
            node_name=cls._make_node_name(target_cls),
            node_class=target_cls,
            node_meta=(),
            edges=edges,
        )

    @classmethod
    def _validate(
        cls, target_cls: type, payload: FacetPayload
    ) -> None:
        cls._validate_no_self_dependency(target_cls)
        cls._validate_deps_are_types(target_cls)

    @classmethod
    def _validate_no_self_dependency(cls, target_cls: type) -> None:
        if target_cls in target_cls._depends_info:
            raise ValueError(
                f"{target_cls.__qualname__} зависит от самого себя."
            )

    @classmethod
    def _validate_deps_are_types(cls, target_cls: type) -> None:
        for dep in target_cls._depends_info:
            if not isinstance(dep, type):
                raise TypeError(
                    f"{target_cls.__qualname__}: зависимость "
                    f"'{dep}' не является типом."
                )
```

## 5. GateCoordinator

```python
from __future__ import annotations
import rustworkx as rx
from .metadata.abstract_gate_host import AbstractGateHost
from .metadata.payload import FacetPayload
from .exceptions import DuplicateNodeError, InvalidGraphError


class GateCoordinator:

    def __init__(self, strict: bool = False) -> None:
        self._strict = strict
        self._gate_hosts: list[type[AbstractGateHost]] = []
        self._registered_gate_hosts: set[type[AbstractGateHost]] = set()
        self._graph = rx.PyDiGraph()
        self._node_index: dict[str, int] = {}
        self._built = False

    # ── Fluent-регистрация ─────────────────────────────────────────────

    def register_gate_host(
        self, gate_host_cls: type[AbstractGateHost]
    ) -> GateCoordinator:
        if self._built:
            raise RuntimeError(
                "Нельзя регистрировать гейтхосты после build()"
            )
        if gate_host_cls in self._registered_gate_hosts:
            raise ValueError(
                f"{gate_host_cls.__name__} уже зарегистрирован"
            )
        self._gate_hosts.append(gate_host_cls)
        self._registered_gate_hosts.add(gate_host_cls)
        return self

    # ── Публичные методы проверки ──────────────────────────────────────

    def has_node(self, node_type: str, name: str) -> bool:
        return self._make_key(node_type, name) in self._node_index

    def node_exists(self, key: str) -> bool:
        return key in self._node_index

    # ── Построение ────────────────────────────────────────────────────

    def build(self) -> GateCoordinator:
        if self._built:
            raise RuntimeError("build() уже был вызван")

        new_payloads = self._collect_payloads()
        self._check_payload_fields(new_payloads)
        self._check_key_uniqueness(new_payloads)
        self._check_referential_integrity(new_payloads)
        self._check_no_cycles(new_payloads)
        self._commit(new_payloads)

        self._built = True
        return self

    # ── Фаза 1: сбор ──────────────────────────────────────────────────

    def _collect_payloads(self) -> list[FacetPayload]:
        payloads = []
        key_to_gate_host: dict[str, str] = {}

        for gh in self._gate_hosts:
            for cls in gh.__subclasses_recursive__():
                payload = gh.inspect(cls)
                if payload is None:
                    continue
                gh._validate(cls, payload)
                key = self._make_key(payload.node_type, payload.node_name)
                if key in key_to_gate_host:
                    raise DuplicateNodeError(
                        f"Конфликт ключа '{key}':\n"
                        f"  создан:    {key_to_gate_host[key]}\n"
                        f"  конфликт:  {gh.__name__}"
                    )
                key_to_gate_host[key] = gh.__name__
                payloads.append(payload)

        return payloads

    # ── Фаза 2: проверки ──────────────────────────────────────────────

    def _check_payload_fields(
        self, payloads: list[FacetPayload]
    ) -> None:
        for p in payloads:
            if not p.node_type:
                raise ValueError(
                    f"node_type пустой в payload для {p.node_class}"
                )
            if not p.node_name:
                raise ValueError(
                    f"node_name пустой в payload для {p.node_class}"
                )
            if not isinstance(p.node_class, type):
                raise TypeError(
                    f"node_class должен быть типом, "
                    f"получен {type(p.node_class)}"
                )

    def _check_key_uniqueness(
        self, payloads: list[FacetPayload]
    ) -> None:
        seen: set[str] = set()
        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            if key in seen:
                raise DuplicateNodeError(
                    f"Дубликат ключа '{key}'"
                )
            seen.add(key)

    def _check_referential_integrity(
        self, payloads: list[FacetPayload]
    ) -> None:
        all_keys = {
            self._make_key(p.node_type, p.node_name)
            for p in payloads
        }
        for p in payloads:
            for edge in p.edges:
                target_key = self._make_key(
                    edge.target_node_type, edge.target_name
                )
                if target_key not in all_keys:
                    raise InvalidGraphError(
                        f"Ребро '{edge.edge_type}' из "
                        f"'{p.node_type}:{p.node_name}' ссылается "
                        f"на несуществующий узел '{target_key}'"
                    )

    def _check_no_cycles(
        self, payloads: list[FacetPayload]
    ) -> None:
        test_graph = rx.PyDiGraph()
        test_index: dict[str, int] = {}

        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            idx = test_graph.add_node(key)
            test_index[key] = idx

        for p in payloads:
            source_key = self._make_key(p.node_type, p.node_name)
            source_idx = test_index[source_key]
            for edge in p.edges:
                if not edge.is_structural:
                    continue
                target_key = self._make_key(
                    edge.target_node_type, edge.target_name
                )
                test_graph.add_edge(
                    source_idx,
                    test_index[target_key],
                    edge.edge_type,
                )

        if not rx.is_directed_acyclic_graph(test_graph):
            raise InvalidGraphError(
                "Структурные рёбра образуют цикл. "
                "Проверьте зависимости depends/connection."
            )

    # ── Фаза 3: коммит ────────────────────────────────────────────────

    def _commit(self, payloads: list[FacetPayload]) -> None:
        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            idx = self._graph.add_node({
                "node_type": p.node_type,
                "name": p.node_name,
                "class_ref": p.node_class,
                "meta": dict(p.node_meta),
            })
            self._node_index[key] = idx

        for p in payloads:
            source_idx = self._node_index[
                self._make_key(p.node_type, p.node_name)
            ]
            for edge in p.edges:
                target_idx = self._node_index[
                    self._make_key(
                        edge.target_node_type, edge.target_name
                    )
                ]
                self._graph.add_edge(
                    source_idx, target_idx,
                    {
                        "edge_type": edge.edge_type,
                        "meta": dict(edge.edge_meta),
                    },
                )

    # ── Утилиты ───────────────────────────────────────────────────────

    @staticmethod
    def _make_key(node_type: str, name: str) -> str:
        return f"{node_type}:{name}"
```

"""
AbstractGateHost — абстрактный базовый класс для всех гейтхостов системы.

═══════════════════════════════════════════════════════════════════════════════
НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════════════════════════════════

AbstractGateHost определяет контракт, которому обязан следовать каждый
гейтхост в системе ActionMachine. Гейтхост — это маркерный миксин,
который одновременно:

1. Разрешает применение соответствующего декоратора к классу
   (issubclass-проверка при декорировании).
2. Знает всех своих наследников через _subclasses_recursive().
3. Умеет инспектировать каждого наследника и собирать данные для графа.

Координатор (GateCoordinator) при build() обходит зарегистрированные
гейтхосты, вызывает inspect() для каждого подкласса, получает
FacetPayload и строит граф.

═══════════════════════════════════════════════════════════════════════════════
ТРИ ОБЯЗАТЕЛЬНЫХ МЕТОДА
═══════════════════════════════════════════════════════════════════════════════

Каждый гейтхост реализует три абстрактных classmethod:

    inspect(target_cls) → FacetPayload | None
        Точка входа. Определяет, подходит ли класс этому гейтхосту.
        Три возможных результата:
        - FacetPayload — класс подходит, данные собраны и провалидированы.
        - None — класс не является субъектом этого гейтхоста.
        - TypeError — класс обязан подойти (наследует гейтхост),
          но данные отсутствуют или повреждены.

    _build_payload(target_cls) → FacetPayload
        Собирает узел и рёбра. Читает атрибуты класса (_role_info,
        _depends_info, _meta_info и т.д.), формирует FacetPayload
        с использованием хелперов базового класса. Не валидирует
        данные — это задача _validate.

    _validate(target_cls, payload) → None
        Оркестратор проверок инвариантов. Вызывает отдельные приватные
        методы для каждого инварианта. Каждый метод — одна проверка,
        одно исключение. Если проверок нет — реализовать явно с pass.

═══════════════════════════════════════════════════════════════════════════════
РАЗДЕЛЕНИЕ ОТВЕТСТВЕННОСТИ
═══════════════════════════════════════════════════════════════════════════════

    inspect()        → проверяет ТОЛЬКО наличие данных (есть _role_info?)
    _build_payload() → читает данные, формирует payload, НЕ валидирует
    _validate()      → проверяет КОРРЕКТНОСТЬ данных и инварианты

Это разделение гарантирует, что:
- inspect() работает быстро — только hasattr/getattr.
- _build_payload() не дублирует проверки из _validate().
- _validate() содержит ВСЕ проверки в одном месте, каждая —
  отдельный метод с отдельным тестом.

═══════════════════════════════════════════════════════════════════════════════

Базовый класс предоставляет четыре хелпера, устраняющие дублирование
в каждом гейтхосте:

    _make_node_name(target_cls, suffix="") → str
        Формирует имя узла "module.ClassName" или "module.ClassName.suffix".
        Префикс типа ("action:", "role:") НЕ добавляется — это делает
        координатор при формировании ключа.

    _make_edge(target_node_type, target_cls, edge_type,
               is_structural, edge_meta=()) → EdgeInfo
        Собирает EdgeInfo без ручного заполнения каждого поля.
        Имя цели формируется через _make_node_name(target_cls).

    _make_edge_by_name(target_node_type, target_name, edge_type,
                       is_structural, edge_meta=()) → EdgeInfo
        Аналог _make_edge для случаев, когда цель — не класс,
        а произвольное строковое имя (например, "context_field:user.user_id").

    _make_meta(**kwargs) → tuple[tuple[str, Any], ...]
        Конвертирует dict-синтаксис в иммутабельный tuple of tuples,
        пригодный для frozen dataclass.

═══════════════════════════════════════════════════════════════════════════════
ОБХОД НАСЛЕДНИКОВ
═══════════════════════════════════════════════════════════════════════════════

Classmethod _subclasses_recursive() рекурсивно собирает всех наследников
гейтхоста через Python-механизм __subclasses__(). Координатор вызывает
этот метод при build() для каждого зарегистрированного гейтхоста.

Python автоматически регистрирует подклассы при определении. Никакой
ручной регистрации классов не существует. Если класс наследует
RoleGateHost — он будет найден через RoleGateHost._subclasses_recursive().

from __future__ import annotations

from abc import ABC, abstractmethod
from typing import Any

from action_machine.metadata.payload import EdgeInfo, FacetPayload


class AbstractGateHost(ABC):
    """
    Абстрактный базовый класс для всех гейтхостов системы ActionMachine.

    Определяет контракт из трёх абстрактных classmethod (inspect,
    _build_payload, _validate) и предоставляет четыре хелпера для
    формирования FacetPayload и EdgeInfo без дублирования кода
    в каждом гейтхосте.

    Все методы — classmethod. Гейтхост не хранит состояния и не
    требует создания экземпляра. Группировка в класс обеспечивает
    пространство имён, наследование хелперов и проверку контракта
    через ABC.

    Координатор (GateCoordinator) при build() вызывает:
    1. cls._subclasses_recursive() — получить всех наследников.
    2. cls.inspect(target_cls) — инспектировать каждого наследника.
    """

    # ═══════════════════════════════════════════════════════════════════
    # Обязательный контракт (три абстрактных метода)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    @abstractmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        """
        Определяет, подходит ли класс этому гейтхосту, и собирает данные.

        Точка входа, вызываемая координатором для каждого подкласса
        гейтхоста. Реализация типичного inspect():

            1. Проверить наличие данных (hasattr/getattr).
            2. Если данных нет → return None.
            3. Если данные обязаны быть, но повреждены → raise TypeError.
            4. Вызвать _build_payload() → FacetPayload.
            5. Вызвать _validate() → проверка инвариантов.
            6. Вернуть payload.

        Аргументы:
            target_cls: класс для инспекции. Гарантированно является
                        подклассом данного гейтхоста (найден через
                        _subclasses_recursive).

        Возвращает:
            FacetPayload — класс подходит, данные собраны.
            None — класс не является субъектом этого гейтхоста
                   (нет соответствующих атрибутов/декораторов).

        Исключения:
            TypeError: класс обязан подойти (наследует гейтхост,
                       имеет частичные данные), но данные повреждены
                       или неполны.
        """
        ...

    @classmethod
    @abstractmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        """
        Собирает FacetPayload из атрибутов класса.

        Читает данные (cls._role_info, cls._depends_info и т.д.),
        формирует узел и рёбра с использованием хелперов базового
        класса: _make_node_name, _make_edge, _make_meta.

        Не валидирует данные — это задача _validate(). Разделение
        гарантирует, что логика сбора и логика проверки не смешиваются.

        Аргументы:
            target_cls: класс, прошедший проверку наличия данных
                        в inspect().

        Возвращает:
            FacetPayload — полное описание узла с рёбрами.
        """
        ...

    @classmethod
    @abstractmethod
    def _validate(cls, target_cls: type, payload: FacetPayload) -> None:
        """
        Оркестратор проверок инвариантов.

        Вызывает отдельные приватные classmethod для каждого инварианта.
        Каждый метод-инвариант выполняет одну проверку и выбрасывает
        одно исключение при нарушении.

        Если у гейтхоста нет инвариантов для проверки (все проверки
        выполнены декоратором при определении класса) — реализовать
        явно с pass:

            @classmethod
            def _validate(cls, target_cls, payload):
                pass  # инварианты проверены декоратором

        Аргументы:
            target_cls: инспектируемый класс.
            payload: FacetPayload, собранный _build_payload().

        Исключения:
            TypeError: нарушение инварианта типа.
            ValueError: нарушение инварианта значения.
        """
        ...

    # ═══════════════════════════════════════════════════════════════════
    # Хелперы для _build_payload (не абстрактные, готовые к использованию)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    def _make_node_name(cls, target_cls: type, suffix: str = "") -> str:
        """
        Формирует имя узла из модуля и имени класса.

        Формат: "module.ClassName" или "module.ClassName.suffix".
        Если модуль "__main__" или отсутствует — только "ClassName".

        Префикс типа ("action:", "role:") НЕ добавляется. Полный ключ
        "node_type:node_name" собирает координатор при коммите.

        Аргументы:
            target_cls: класс, для которого формируется имя.
            suffix: опциональный суффикс, добавляемый через точку.
                    Используется для дочерних узлов: аспектов,
                    чекеров, полей сущностей.
                    Пустая строка — суффикс не добавляется.

        Возвращает:
            str — имя узла.

        Примеры:
            _make_node_name(CreateOrderAction)
            → "myapp.orders.CreateOrderAction"

            _make_node_name(CreateOrderAction, "validate_aspect")
            → "myapp.orders.CreateOrderAction.validate_aspect"

            _make_node_name(OrderEntity, "amount")
            → "myapp.domain.OrderEntity.amount"
        """
        module = getattr(target_cls, "__module__", None)
        if module and module != "__main__":
            name = f"{module}.{target_cls.__qualname__}"
        else:
            name = target_cls.__qualname__
        if suffix:
            return f"{name}.{suffix}"
        return name

    @classmethod
    def _make_edge(
        cls,
        target_node_type: str,
        target_cls: type,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с автоматическим формированием имени цели.

        Имя целевого узла формируется через _make_node_name(target_cls).
        Используется когда цель ребра — класс Python.

        Аргументы:
            target_node_type: тип целевого узла ("action", "entity",
                              "domain", "dependency" и т.д.).
            target_cls: класс, являющийся целью ребра.
            edge_type: тип ребра ("depends", "connection", "has_aspect",
                       "belongs_to" и т.д.).
            is_structural: True — структурное ребро (циклы запрещены).
                           False — информационное ребро (циклы допустимы).
            edge_meta: дополнительные метаданные ребра.
                       Пустой tuple по умолчанию.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge(
                target_node_type="dependency",
                target_cls=PaymentService,
                edge_type="depends",
                is_structural=True,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=cls._make_node_name(target_cls),
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_edge_by_name(
        cls,
        target_node_type: str,
        target_name: str,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с произвольным строковым именем цели.

        Используется когда цель ребра — не класс Python, а строковый
        идентификатор. Например, узлы контекстных полей
        ("context_field:user.user_id") или доменов ("domain:orders").

        Аргументы:
            target_node_type: тип целевого узла.
            target_name: строковое имя целевого узла.
            edge_type: тип ребра.
            is_structural: структурное или информационное.
            edge_meta: дополнительные метаданные ребра.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge_by_name(
                target_node_type="context_field",
                target_name="user.user_id",
                edge_type="requires_context",
                is_structural=False,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=target_name,
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_meta(cls, **kwargs: Any) -> tuple[tuple[str, Any], ...]:
        """
        Конвертирует именованные аргументы в иммутабельный tuple of tuples.

        Frozen dataclass требует хешируемые поля. dict не хешируем.
        Этот хелпер позволяет гейтхостам использовать удобный
        dict-синтаксис (kwargs) при создании метаданных, получая
        на выходе иммутабельную структуру.

        Координатор конвертирует tuple of tuples обратно в dict
        при коммите: dict(node_meta).

        Аргументы:
            **kwargs: произвольные пары ключ-значение метаданных.

        Возвращает:
            tuple[tuple[str, Any], ...] — иммутабельные метаданные.

        Пример:
            cls._make_meta(spec="admin", description="Администратор")
            → (("spec", "admin"), ("description", "Администратор"))

            cls._make_meta()
            → ()
        """
        return tuple(kwargs.items())

        Использует Python-механизм __subclasses__(), который
        автоматически регистрирует подклассы при определении.
        Никакой ручной регистрации не требуется.

        Координатор вызывает этот метод при build() для каждого
        зарегистрированного гейтхоста, чтобы получить список
        всех классов, которые нужно инспектировать.

        Возвращает:
            list[type] — все прямые и транзитивные подклассы
                         данного гейтхоста. Порядок: depth-first,
                         прямые подклассы перед их потомками.

        Пример:
            class RoleGateHost(AbstractGateHost): ...
            class BaseAction(RoleGateHost): ...
            class CreateOrderAction(BaseAction): ...
            class UpdateOrderAction(BaseAction): ...

            RoleGateHost._subclasses_recursive()
            → [BaseAction, CreateOrderAction, UpdateOrderAction]
        """
        result: list[type] = []
        for subclass in cls.__subclasses__():
            result.append(subclass)
            if hasattr(subclass, "_subclasses_recursive"):
                result.extend(subclass._subclasses_recursive())
        return result

class DuplicateNodeError(ValueError):
    """
    Два гейтхоста создали узлы с одинаковым ключом в графе.

    Выбрасывается координатором на фазе 1 (сбор payload) при
    обнаружении конфликта ключей "node_type:node_name". Сообщение
    содержит полный ключ и имена обоих гейтхостов, чтобы разработчик
    мог быстро определить источник конфликта.

    Наследует ValueError, потому что это ошибка конфигурации системы:
    два гейтхоста претендуют на один и тот же узел графа.

    - Два гейтхоста используют одинаковый node_type для одного класса.
    - Гейтхост создаёт дочерний узел с именем, совпадающим с узлом

        "Конфликт ключа 'action:module.CreateOrderAction':

        first_gate_host : str

        second_gate_host : str
            Имя гейтхоста, обнаружившего конфликт.
    """

    def __init__(
        self,
        key: str,
        first_gate_host: str,
        second_gate_host: str,
    ) -> None:

        Инициализирует исключение.

        Аргументы:
            key: конфликтующий ключ "node_type:node_name".
            first_gate_host: имя гейтхоста, создавшего узел первым.
            second_gate_host: имя гейтхоста, обнаружившего конфликт.

class InvalidGraphError(Exception):
    """
    Нарушение структурной целостности графа.

    Выбрасывается координатором на фазе 2 (проверки) в двух случаях:

    1. Ссылочная целостность — ребро ссылается на узел, которого нет

    ацикличность — циклические связи между сущностями (Order ↔ Customer)

    Наследует Exception (не ValueError и не TypeError), потому что
    это ошибка структуры графа — категория, отличная от ошибок
    значений или типов.
    """

    pass


class PayloadValidationError(TypeError):
    """
    Некорректные данные в FacetPayload.

    Выбрасывается координатором на фазе 2 при проверке обязательных
    полей payload, собранных гейтхостами. Каждый payload проверяется:

    - node_type — непустая строка.
    - node_name — непустая строка.
    - node_class — экземпляр type (класс Python).

    Наследует TypeError, потому что все три проверки по сути являются
    проверками типа: пустая строка — не валидный идентификатор,

    Типичные причины:
    - Гейтхост забыл указать node_type в _build_payload().
    - _make_node_name() получил класс без __module__.
    - В node_class передан экземпляр вместо класса.

    Атрибуты:
        node_class : type | object
            Класс (или объект), для которого payload невалиден.
        field_name : str
            Имя поля payload, вызвавшего ошибку.
        detail : str
            Описание проблемы.
    """

    def __init__(
        self,
        node_class: object,
        field_name: str,
        detail: str,
    ) -> None:
        """
        Инициализирует исключение.

        Аргументы:
            node_class: класс или объект из payload.node_class.
            field_name: имя поля ("node_type", "node_name", "node_class").
            detail: описание проблемы.
        """
        self.node_class: object = node_class
        self.field_name: str = field_name
        self.detail: str = detail

        class_name = (
            node_class.__qualname__
            if isinstance(node_class, type)
            else repr(node_class)
        )
        super().__init__(
            f"Невалидный payload для {class_name}: "
            f"поле '{field_name}' — {detail}"
        )

"""
BaseGateHostInspector — абстрактный базовый класс для всех гейтхостов системы.

BaseGateHostInspector определяет контракт, которому обязан следовать каждый
гейтхост в системе ActionMachine. Гейтхост — это маркерный миксин,
который одновременно:

1. Разрешает применение соответствующего декоратора к классу
   (issubclass-проверка при декорировании).
2. Знает всех своих наследников через _subclasses_recursive().
3. Умеет инспектировать каждого наследника и собирать данные для графа.

Координатор (GateCoordinator) при build() обходит зарегистрированные
гейтхосты, вызывает inspect() для каждого подкласса, получает
FacetPayload и строит граф.

═══════════════════════════════════════════════════════════════════════════════
ТРИ ОБЯЗАТЕЛЬНЫХ МЕТОДА
═══════════════════════════════════════════════════════════════════════════════

Каждый гейтхост реализует три абстрактных classmethod:

    inspect(target_cls) → FacetPayload | None
        Точка входа. Определяет, подходит ли класс этому гейтхосту.
        Три возможных результата:
        - FacetPayload — класс подходит, данные собраны и провалидированы.
        - None — класс не является субъектом этого гейтхоста.
        - TypeError — класс обязан подойти (наследует гейтхост),
          но данные отсутствуют или повреждены.

    _build_payload(target_cls) → FacetPayload
        Собирает узел и рёбра. Читает атрибуты класса (_role_info,
        _depends_info, _meta_info и т.д.), формирует FacetPayload
        с использованием хелперов базового класса. Не валидирует
        данные — это задача _validate.

    _validate(target_cls, payload) → None
        Оркестратор проверок инвариантов. Вызывает отдельные приватные
        методы для каждого инварианта. Каждый метод — одна проверка,
        одно исключение. Если проверок нет — реализовать явно с pass.

═══════════════════════════════════════════════════════════════════════════════
РАЗДЕЛЕНИЕ ОТВЕТСТВЕННОСТИ
═══════════════════════════════════════════════════════════════════════════════

    inspect()        → проверяет ТОЛЬКО наличие данных (есть _role_info?)
    _build_payload() → читает данные, формирует payload, НЕ валидирует
    _validate()      → проверяет КОРРЕКТНОСТЬ данных и инварианты

Это разделение гарантирует, что:
- inspect() работает быстро — только hasattr/getattr.
- _build_payload() не дублирует проверки из _validate().
- _validate() содержит ВСЕ проверки в одном месте, каждая —
  отдельный метод с отдельным тестом.

ХЕЛПЕРЫ
═══════════════════════════════════════════════════════════════════════════════

Базовый класс предоставляет четыре хелпера, устраняющие дублирование
в каждом гейтхосте:

    _make_node_name(target_cls, suffix="") → str
        Формирует имя узла "module.ClassName" или "module.ClassName.suffix".
        Префикс типа ("action:", "role:") НЕ добавляется — это делает
        координатор при формировании ключа.

    _make_edge(target_node_type, target_cls, edge_type,
               is_structural, edge_meta=()) → EdgeInfo
        Собирает EdgeInfo без ручного заполнения каждого поля.
        Имя цели формируется через _make_node_name(target_cls).

    _make_edge_by_name(target_node_type, target_name, edge_type,
                       is_structural, edge_meta=()) → EdgeInfo
        Аналог _make_edge для случаев, когда цель — не класс,
        а произвольное строковое имя (например, "context_field:user.user_id").

    _make_meta(**kwargs) → tuple[tuple[str, Any], ...]
        Конвертирует dict-синтаксис в иммутабельный tuple of tuples,
        пригодный для frozen dataclass.

═══════════════════════════════════════════════════════════════════════════════
ОБХОД НАСЛЕДНИКОВ
═══════════════════════════════════════════════════════════════════════════════

Classmethod _subclasses_recursive() рекурсивно собирает всех наследников
гейтхоста через Python-механизм __subclasses__(). Координатор вызывает
этот метод при build() для каждого зарегистрированного гейтхоста.

Python автоматически регистрирует подклассы при определении. Никакой
ручной регистрации классов не существует. Если класс наследует
RoleGateHost — он будет найден через RoleGateHost._subclasses_recursive().

    class RoleGateHost(BaseGateHostInspector):

        @classmethod

            role_info = getattr(target_cls, "_role_info", None)
            if role_info is None:

        @classmethod
        def _validate(cls, target_cls: type, payload: FacetPayload) -> None:

from __future__ import annotations

from abc import ABC, abstractmethod
from typing import Any

from action_machine.metadata.payload import EdgeInfo, FacetPayload


class BaseGateHostInspector(ABC):
    """
    Абстрактный базовый класс для всех гейтхостов системы ActionMachine.

    Определяет контракт из трёх абстрактных classmethod (inspect,
    _build_payload, _validate) и предоставляет четыре хелпера для
    формирования FacetPayload и EdgeInfo без дублирования кода
    в каждом гейтхосте.

    Все методы — classmethod. Гейтхост не хранит состояния и не
    требует создания экземпляра. Группировка в класс обеспечивает
    пространство имён, наследование хелперов и проверку контракта
    через ABC.

    Координатор (GateCoordinator) при build() вызывает:
    1. cls._subclasses_recursive() — получить всех наследников.
    2. cls.inspect(target_cls) — инспектировать каждого наследника.
    """

    # ═══════════════════════════════════════════════════════════════════
    # Обязательный контракт (три абстрактных метода)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    @abstractmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        """
        Определяет, подходит ли класс этому гейтхосту, и собирает данные.

        Точка входа, вызываемая координатором для каждого подкласса
        гейтхоста. Реализация типичного inspect():

            1. Проверить наличие данных (hasattr/getattr).
            2. Если данных нет → return None.
            3. Если данные обязаны быть, но повреждены → raise TypeError.
            4. Вызвать _build_payload() → FacetPayload.
            5. Вызвать _validate() → проверка инвариантов.
            6. Вернуть payload.

        Аргументы:
            target_cls: класс для инспекции. Гарантированно является
                        подклассом данного гейтхоста (найден через
                        _subclasses_recursive).

        Возвращает:
            FacetPayload — класс подходит, данные собраны.
            None — класс не является субъектом этого гейтхоста
                   (нет соответствующих атрибутов/декораторов).

        Исключения:
            TypeError: класс обязан подойти (наследует гейтхост,
                       имеет частичные данные), но данные повреждены
                       или неполны.
        """
        ...

    @classmethod
    @abstractmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        """
        Собирает FacetPayload из атрибутов класса.

        Читает данные (cls._role_info, cls._depends_info и т.д.),
        формирует узел и рёбра с использованием хелперов базового
        класса: _make_node_name, _make_edge, _make_meta.

        Не валидирует данные — это задача _validate(). Разделение
        гарантирует, что логика сбора и логика проверки не смешиваются.

        Аргументы:
            target_cls: класс, прошедший проверку наличия данных
                        в inspect().

        Возвращает:
            FacetPayload — полное описание узла с рёбрами.
        """
        ...

    @classmethod
    @abstractmethod
    def _validate(cls, target_cls: type, payload: FacetPayload) -> None:
        """
        Оркестратор проверок инвариантов.

        Вызывает отдельные приватные classmethod для каждого инварианта.
        Каждый метод-инвариант выполняет одну проверку и выбрасывает
        одно исключение при нарушении.

        Если у гейтхоста нет инвариантов для проверки (все проверки
        выполнены декоратором при определении класса) — реализовать
        явно с pass:

            @classmethod
            def _validate(cls, target_cls, payload):
                pass  # инварианты проверены декоратором

        Аргументы:
            target_cls: инспектируемый класс.
            payload: FacetPayload, собранный _build_payload().

        Исключения:
            TypeError: нарушение инварианта типа.
            ValueError: нарушение инварианта значения.
        """
        ...

    # ═══════════════════════════════════════════════════════════════════
    # Хелперы для _build_payload (не абстрактные, готовые к использованию)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    def _make_node_name(cls, target_cls: type, suffix: str = "") -> str:
        """
        Формирует имя узла из модуля и имени класса.

        Формат: "module.ClassName" или "module.ClassName.suffix".
        Если модуль "__main__" или отсутствует — только "ClassName".

        Префикс типа ("action:", "role:") НЕ добавляется. Полный ключ
        "node_type:node_name" собирает координатор при коммите.

        Аргументы:
            target_cls: класс, для которого формируется имя.
            suffix: опциональный суффикс, добавляемый через точку.
                    Используется для дочерних узлов: аспектов,
                    чекеров, полей сущностей.
                    Пустая строка — суффикс не добавляется.

        Возвращает:
            str — имя узла.

        Примеры:
            _make_node_name(CreateOrderAction)
            → "myapp.orders.CreateOrderAction"

            _make_node_name(CreateOrderAction, "validate_aspect")
            → "myapp.orders.CreateOrderAction.validate_aspect"

            _make_node_name(OrderEntity, "amount")
            → "myapp.domain.OrderEntity.amount"
        """
        module = getattr(target_cls, "__module__", None)
        if module and module != "__main__":
            name = f"{module}.{target_cls.__qualname__}"
        else:
            name = target_cls.__qualname__
        if suffix:
            return f"{name}.{suffix}"
        return name

    @classmethod
    def _make_edge(
        cls,
        target_node_type: str,
        target_cls: type,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с автоматическим формированием имени цели.

        Имя целевого узла формируется через _make_node_name(target_cls).
        Используется когда цель ребра — класс Python.

        Аргументы:
            target_node_type: тип целевого узла ("action", "entity",
                              "domain", "dependency" и т.д.).
            target_cls: класс, являющийся целью ребра.
            edge_type: тип ребра ("depends", "connection", "has_aspect",
                       "belongs_to" и т.д.).
            is_structural: True — структурное ребро (циклы запрещены).
                           False — информационное ребро (циклы допустимы).
            edge_meta: дополнительные метаданные ребра.
                       Пустой tuple по умолчанию.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge(
                target_node_type="dependency",
                target_cls=PaymentService,
                edge_type="depends",
                is_structural=True,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=cls._make_node_name(target_cls),
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_edge_by_name(
        cls,
        target_node_type: str,
        target_name: str,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с произвольным строковым именем цели.

        Используется когда цель ребра — не класс Python, а строковый
        идентификатор. Например, узлы контекстных полей
        ("context_field:user.user_id") или доменов ("domain:orders").

        Аргументы:
            target_node_type: тип целевого узла.
            target_name: строковое имя целевого узла.
            edge_type: тип ребра.
            is_structural: структурное или информационное.
            edge_meta: дополнительные метаданные ребра.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge_by_name(
                target_node_type="context_field",
                target_name="user.user_id",
                edge_type="requires_context",
                is_structural=False,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=target_name,
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_meta(cls, **kwargs: Any) -> tuple[tuple[str, Any], ...]:
        """
        Конвертирует именованные аргументы в иммутабельный tuple of tuples.

        Frozen dataclass требует хешируемые поля. dict не хешируем.
        Этот хелпер позволяет гейтхостам использовать удобный
        dict-синтаксис (kwargs) при создании метаданных, получая
        на выходе иммутабельную структуру.

        Координатор конвертирует tuple of tuples обратно в dict
        при коммите: dict(node_meta).

        Аргументы:
            **kwargs: произвольные пары ключ-значение метаданных.

        Возвращает:
            tuple[tuple[str, Any], ...] — иммутабельные метаданные.

        Пример:
            cls._make_meta(spec="admin", description="Администратор")
            → (("spec", "admin"), ("description", "Администратор"))

            cls._make_meta()
            → ()
        """
        return tuple(kwargs.items())

    # ═══════════════════════════════════════════════════════════════════
    # Обход наследников
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        """
        Рекурсивно собирает всех наследников гейтхоста.

        Использует Python-механизм __subclasses__(), который
        автоматически регистрирует подклассы при определении.
        Никакой ручной регистрации не требуется.

        Координатор вызывает этот метод при build() для каждого
        зарегистрированного гейтхоста, чтобы получить список
        всех классов, которые нужно инспектировать.

        Возвращает:
            list[type] — все прямые и транзитивные подклассы
                         данного гейтхоста. Порядок: depth-first,
                         прямые подклассы перед их потомками.

        Пример:
            class RoleGateHost(BaseGateHostInspector): ...
            class BaseAction(RoleGateHost): ...
            class CreateOrderAction(BaseAction): ...
            class UpdateOrderAction(BaseAction): ...

            RoleGateHost._subclasses_recursive()
            → [BaseAction, CreateOrderAction, UpdateOrderAction]
        """
        result: list[type] = []
        for subclass in cls.__subclasses__():
            result.append(subclass)
            if hasattr(subclass, "_subclasses_recursive"):
                result.extend(subclass._subclasses_recursive())
        return result

def _collect_subclasses(mixin: type) -> list[type]:
    """
    Рекурсивно собирает всех наследников маркерного миксина.

    Обходит дерево наследования через Python-механизм __subclasses__().
    Порядок: depth-first — прямые подклассы перед их потомками.

    Вынесена как модульная функция (а не метод класса), потому что
    используется одинаково во всех инспекторах. Каждый инспектор
    вызывает _collect_subclasses(cls._target_mixin) в своём
    _subclasses_recursive().

    Аргументы:
        mixin: маркерный миксин, наследников которого нужно собрать.
               Например, RoleGateHost, AspectGateHost.

    Возвращает:
        list[type] — все прямые и транзитивные подклассы миксина.
    """
    result: list[type] = []
    for sub in mixin.__subclasses__():
        result.append(sub)
        result.extend(_collect_subclasses(sub))
    return result


class RoleGateHostInspector(BaseGateHostInspector):
    """
    Инспектор гейтхоста ролей.

    Обходит наследников RoleGateHost, обнаруживает классы с декоратором
    @check_roles и собирает FacetPayload с ролевой спецификацией
    для графа координатора.

    Узел в графе: тип "role", без исходящих рёбер.

Финальный план рефакторинга GateCoordinator
Гейтхосты-инспекторы с транзакционным построением графа
Контекст
Неинвазивная трансформация. Существующие маркерные миксины, декораторы и классы остаются на месте без изменений. Добавляются новые файлы — инспекторы гейтхостов. Пользовательский код не меняется. Старый MetadataBuilder работает параллельно до полного перехода, после чего удаляется.

1. Архитектура
1.1. Центральная идея
Система строит граф зависимостей между объектами предметной области. Узлы графа — это классы (Action, Entity, Role, Aspect...). Рёбра — отношения между ними (depends, has_aspect, belongs_to...).

Copy        CreateOrderAction
         /            \
    depends          has_aspect
       /                  \
 PaymentService       validate_aspect
       \
    belongs_to
          \
        OrdersDomain
Граф строится один раз при старте приложения через coordinator.build(). После построения — только читается.

1.2. Два слоя: маркеры и инспекторы
Каждый гейтхост существует в двух ипостасях:

Маркерный миксин (RoleGateHost, AspectGateHost, DependencyGateHost[T]...)

Живёт в MRO класса BaseAction (или BaseEntity, BaseResourceManager).
Разрешает применение соответствующего декоратора через issubclass-проверку.
Не содержит логики инспекции.
Не наследует BaseGateHostInspector.
Не меняется при рефакторинге.
Инспектор (RoleGateHostInspector, AspectGateHostInspector, DependencyGateHostInspector...)

Наследует BaseGateHostInspector.
Реализует inspect() и _build_payload().
Обходит наследников маркера через _target_mixin.
Регистрируется в координаторе.
Новый файл, лежит рядом с маркером.
Связь между ними — поле _target_mixin инспектора. Инспектор знает, наследников какого маркера обходить. Маркер не знает про инспектор.

│  RoleGateHost           │         │  RoleGateHostInspector       │
│  (маркерный миксин)     │◄────────│  (наследник BaseGateHost-    │
│                         │ _target │   Inspector)                 │
│  Живёт в BaseAction MRO │ _mixin  │                              │
│  Разрешает @check_roles │         │  inspect() → FacetPayload    │
│  НЕ МЕНЯЕТСЯ            │         │  _build_payload() → payload  │
└─────────────────────────┘         └──────────────────────────────┘
1.3. Кто знает про классы?
Python автоматически регистрирует подклассы при определении через __subclasses__(). Никакой ручной регистрации классов не существует.

Инспектор обходит наследников своего маркера:

Copyclass RoleGateHostInspector(BaseGateHostInspector):
    _target_mixin = RoleGateHost

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        return _collect_subclasses(cls._target_mixin)

AspectGateHost.__subclasses__() (рекурсивно)

EntityGateHost.__subclasses__() (рекурсивно)
    → [BaseEntity, OrderEntity, PaymentEntity, ...]
Один класс может быть обнаружен несколькими инспекторами. Например, CreateOrderAction найдётся и RoleGateHostInspector, и AspectGateHostInspector, и DependencyGateHostInspector. Каждый создаёт свой payload с уникальным node_type.

1.4. Разделение валидации
Валидация выполняется на двух уровнях, каждый — в одном месте:

Декораторы (@check_roles, @regular_aspect, @depends...)

Проверяют аргументы при import-time.
Типы, пустоту, issubclass, дубликаты.
Обнаруживают ошибки немедленно при определении класса.
Не меняются при рефакторинге.
Координатор (GateCoordinator.build())

Глобальные структурные проверки после сбора всех payload.
Уникальность ключей узлов.
Ссылочная целостность рёбер (цель существует).
Ацикличность структурных рёбер.
Обязательные поля payload непустые.
Одно место для всех проверок графа.
_validate() отсутствует в контракте инспектора. Логика проверки не размазывается между декоратором и инспектором. Декоратор отвечает за свои аргументы, координатор — за целостность графа.

1.13. Утилита _collect_subclasses
Функция _collect_subclasses(mixin) рекурсивно собирает всех наследников маркерного миксина. Используется каждым инспектором в _subclasses_recursive(). Вынесена в metadata/utils.py:

Copy# metadata/utils.py
def collect_subclasses(mixin: type) -> list[type]:
    result: list[type] = []
    for sub in mixin.__subclasses__():
        result.append(sub)
        result.extend(collect_subclasses(sub))
    return result

4. Примеры реализации инспекторов
RoleGateHostInspector — без рёбер
Copyclass RoleGateHostInspector(BaseGateHostInspector):
    _target_mixin: type = RoleGateHost

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        return collect_subclasses(cls._target_mixin)

    @classmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        role_info = getattr(target_cls, "_role_info", None)
        if role_info is None:
            return None
        return cls._build_payload(target_cls)

    @classmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        return FacetPayload(
            node_type="role",
            node_name=cls._make_node_name(target_cls),
            node_class=target_cls,
            node_meta=cls._make_meta(
                spec=target_cls._role_info["spec"]
            ),
        )
DependencyGateHostInspector — со структурными рёбрами
Copyclass DependencyGateHostInspector(BaseGateHostInspector):
    _target_mixin: type = DependencyGateHost

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        return collect_subclasses(cls._target_mixin)

    @classmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        depends_info = getattr(target_cls, "_depends_info", None)
        if not depends_info:
            return None
        return cls._build_payload(target_cls)

    @classmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        edges = tuple(
            cls._make_edge(
                target_node_type="dependency",
                target_cls=dep_info.cls,
                edge_type="depends",
                is_structural=True,
            )
            for dep_info in target_cls._depends_info
        )
        return FacetPayload(
            node_type="action",
            node_name=cls._make_node_name(target_cls),
            node_class=target_cls,
            edges=edges,
        )

5. GateCoordinator
Copyclass GateCoordinator:

    def __init__(self, strict: bool = False) -> None:
        self._strict = strict
        self._inspectors: list[type[BaseGateHostInspector]] = []
        self._registered: set[type[BaseGateHostInspector]] = set()
        self._graph = rx.PyDiGraph()
        self._node_index: dict[str, int] = {}
        self._built = False

    def register(self, inspector_cls) -> GateCoordinator:
        """Fluent-регистрация инспектора."""
        if self._built:
            raise RuntimeError("Регистрация после build() запрещена")
        if inspector_cls in self._registered:
            raise ValueError(f"{inspector_cls.__name__} уже зарегистрирован")
        self._inspectors.append(inspector_cls)
        self._registered.add(inspector_cls)
        return self

    def build(self) -> GateCoordinator:
        """Транзакционное построение графа."""
        if self._built:
            raise RuntimeError("build() уже вызван")
        all_payloads = self._phase1_collect()
        self._phase2_validate(all_payloads)
        self._phase3_commit(all_payloads)
        self._built = True
        return self

    def _phase1_collect(self) -> list[FacetPayload]: ...
    def _phase2_validate(self, payloads) -> None: ...
    def _phase3_commit(self, payloads) -> None: ...

    # Рантайм-доступ к данным графа:
    def get_role_spec(self, cls) -> str | list[str]: ...
    def get_regular_aspects(self, cls) -> tuple[AspectMeta, ...]: ...

    def get_checkers_for_aspect(self, cls, name) -> tuple[CheckerMeta, ...]: ...

НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════════════════════════════════

BaseGateHostInspector определяет контракт, которому обязан следовать каждый
гейтхост в системе ActionMachine. Гейтхост — это маркерный миксин,
который одновременно:

1. Разрешает применение соответствующего декоратора к классу
   (issubclass-проверка при декорировании).
2. Знает всех своих наследников через _subclasses_recursive().
3. Умеет инспектировать каждого наследника и собирать данные для графа.

Координатор (GateCoordinator) при build() обходит зарегистрированные
гейтхосты, вызывает inspect() для каждого подкласса, получает
FacetPayload и строит граф.

═══════════════════════════════════════════════════════════════════════════════
ТРИ ОБЯЗАТЕЛЬНЫХ МЕТОДА
═══════════════════════════════════════════════════════════════════════════════

Каждый гейтхост реализует три абстрактных classmethod:

    inspect(target_cls) → FacetPayload | None
        Точка входа. Определяет, подходит ли класс этому гейтхосту.
        Три возможных результата:
        - FacetPayload — класс подходит, данные собраны и провалидированы.
        - None — класс не является субъектом этого гейтхоста.
        - TypeError — класс обязан подойти (наследует гейтхост),
          но данные отсутствуют или повреждены.

    _build_payload(target_cls) → FacetPayload
        Собирает узел и рёбра. Читает атрибуты класса (_role_info,
        _depends_info, _meta_info и т.д.), формирует FacetPayload
        с использованием хелперов базового класса. Не валидирует
        данные — это задача _validate.

═══════════════════════════════════════════════════════════════════════════════
РАЗДЕЛЕНИЕ ОТВЕТСТВЕННОСТИ
═══════════════════════════════════════════════════════════════════════════════

    inspect()        → проверяет ТОЛЬКО наличие данных (есть _role_info?)
    _build_payload() → читает данные, формирует payload, НЕ валидирует

Это разделение гарантирует, что:
- inspect() работает быстро — только hasattr/getattr.
- _build_payload() не дублирует проверки из _validate().

        Конвертирует dict-синтаксис в иммутабельный tuple of tuples,

from abc import ABC, abstractmethod
from typing import Any

from action_machine.metadata.payload import EdgeInfo, FacetPayload


class BaseGateHostInspector(ABC):
    """
    Абстрактный базовый класс для всех гейтхостов системы ActionMachine.

    Определяет контракт из трёх абстрактных classmethod (inspect,
    _build_payload, _validate) и предоставляет четыре хелпера для
    формирования FacetPayload и EdgeInfo без дублирования кода
    в каждом гейтхосте.

    Все методы — classmethod. Гейтхост не хранит состояния и не
    требует создания экземпляра. Группировка в класс обеспечивает
    пространство имён, наследование хелперов и проверку контракта
    через ABC.

    Координатор (GateCoordinator) при build() вызывает:
    1. cls._subclasses_recursive() — получить всех наследников.
    2. cls.inspect(target_cls) — инспектировать каждого наследника.
    """

    # ═══════════════════════════════════════════════════════════════════
    # Обязательный контракт (три абстрактных метода)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    @abstractmethod
    def inspect(cls, target_cls: type) -> FacetPayload | None:
        """
        Определяет, подходит ли класс этому гейтхосту, и собирает данные.

        Точка входа, вызываемая координатором для каждого подкласса
        гейтхоста. Реализация типичного inspect():

            1. Проверить наличие данных (hasattr/getattr).
            2. Если данных нет → return None.
            3. Если данные обязаны быть, но повреждены → raise TypeError.
            4. Вызвать _build_payload() → FacetPayload.
            5. Вызвать _validate() → проверка инвариантов.
            6. Вернуть payload.

        Аргументы:
            target_cls: класс для инспекции. Гарантированно является
                        подклассом данного гейтхоста (найден через
                        _subclasses_recursive).

        Возвращает:
            FacetPayload — класс подходит, данные собраны.
            None — класс не является субъектом этого гейтхоста
                   (нет соответствующих атрибутов/декораторов).

        Исключения:
            TypeError: класс обязан подойти (наследует гейтхост,
                       имеет частичные данные), но данные повреждены
                       или неполны.
        """
        ...

    @classmethod
    @abstractmethod
    def _build_payload(cls, target_cls: type) -> FacetPayload:
        """
        Собирает FacetPayload из атрибутов класса.

        Читает данные (cls._role_info, cls._depends_info и т.д.),
        формирует узел и рёбра с использованием хелперов базового
        класса: _make_node_name, _make_edge, _make_meta.

        Не валидирует данные — это задача _validate(). Разделение
        гарантирует, что логика сбора и логика проверки не смешиваются.

        Аргументы:
            target_cls: класс, прошедший проверку наличия данных
                        в inspect().

        Возвращает:
            FacetPayload — полное описание узла с рёбрами.
        """
        ...

    # ═══════════════════════════════════════════════════════════════════
    # Хелперы для _build_payload (не абстрактные, готовые к использованию)
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    def _make_node_name(cls, target_cls: type, suffix: str = "") -> str:
        """
        Формирует имя узла из модуля и имени класса.

        Формат: "module.ClassName" или "module.ClassName.suffix".
        Если модуль "__main__" или отсутствует — только "ClassName".

        Префикс типа ("action:", "role:") НЕ добавляется. Полный ключ
        "node_type:node_name" собирает координатор при коммите.

        Аргументы:
            target_cls: класс, для которого формируется имя.
            suffix: опциональный суффикс, добавляемый через точку.
                    Используется для дочерних узлов: аспектов,
                    чекеров, полей сущностей.
                    Пустая строка — суффикс не добавляется.

        Возвращает:
            str — имя узла.

        Примеры:
            _make_node_name(CreateOrderAction)
            → "myapp.orders.CreateOrderAction"

            _make_node_name(CreateOrderAction, "validate_aspect")
            → "myapp.orders.CreateOrderAction.validate_aspect"

            _make_node_name(OrderEntity, "amount")
            → "myapp.domain.OrderEntity.amount"
        """
        module = getattr(target_cls, "__module__", None)
        if module and module != "__main__":
            name = f"{module}.{target_cls.__qualname__}"
        else:
            name = target_cls.__qualname__
        if suffix:
            return f"{name}.{suffix}"
        return name

    @classmethod
    def _make_edge(
        cls,
        target_node_type: str,
        target_cls: type,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с автоматическим формированием имени цели.

        Имя целевого узла формируется через _make_node_name(target_cls).
        Используется когда цель ребра — класс Python.

        Аргументы:
            target_node_type: тип целевого узла ("action", "entity",
                              "domain", "dependency" и т.д.).
            target_cls: класс, являющийся целью ребра.
            edge_type: тип ребра ("depends", "connection", "has_aspect",
                       "belongs_to" и т.д.).
            is_structural: True — структурное ребро (циклы запрещены).
                           False — информационное ребро (циклы допустимы).
            edge_meta: дополнительные метаданные ребра.
                       Пустой tuple по умолчанию.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge(
                target_node_type="dependency",
                target_cls=PaymentService,
                edge_type="depends",
                is_structural=True,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=cls._make_node_name(target_cls),
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_edge_by_name(
        cls,
        target_node_type: str,
        target_name: str,
        edge_type: str,
        is_structural: bool,
        edge_meta: tuple[tuple[str, Any], ...] = (),
    ) -> EdgeInfo:
        """
        Собирает EdgeInfo с произвольным строковым именем цели.

        Используется когда цель ребра — не класс Python, а строковый
        идентификатор. Например, узлы контекстных полей
        ("context_field:user.user_id") или доменов ("domain:orders").

        Аргументы:
            target_node_type: тип целевого узла.
            target_name: строковое имя целевого узла.
            edge_type: тип ребра.
            is_structural: структурное или информационное.
            edge_meta: дополнительные метаданные ребра.

        Возвращает:
            EdgeInfo — описание ребра.

        Пример:
            cls._make_edge_by_name(
                target_node_type="context_field",
                target_name="user.user_id",
                edge_type="requires_context",
                is_structural=False,
            )
        """
        return EdgeInfo(
            target_node_type=target_node_type,
            target_name=target_name,
            edge_type=edge_type,
            is_structural=is_structural,
            edge_meta=edge_meta,
        )

    @classmethod
    def _make_meta(cls, **kwargs: Any) -> tuple[tuple[str, Any], ...]:
        """
        Конвертирует именованные аргументы в иммутабельный tuple of tuples.

        Frozen dataclass требует хешируемые поля. dict не хешируем.
        Этот хелпер позволяет гейтхостам использовать удобный
        dict-синтаксис (kwargs) при создании метаданных, получая
        на выходе иммутабельную структуру.

        Координатор конвертирует tuple of tuples обратно в dict
        при коммите: dict(node_meta).

        Аргументы:
            **kwargs: произвольные пары ключ-значение метаданных.

        Возвращает:
            tuple[tuple[str, Any], ...] — иммутабельные метаданные.

        Пример:
            cls._make_meta(spec="admin", description="Администратор")
            → (("spec", "admin"), ("description", "Администратор"))

            cls._make_meta()
            → ()
        """
        return tuple(kwargs.items())

    # ═══════════════════════════════════════════════════════════════════
    # Обход наследников
    # ═══════════════════════════════════════════════════════════════════

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        """
        Рекурсивно собирает всех наследников гейтхоста.

        Использует Python-механизм __subclasses__(), который
        автоматически регистрирует подклассы при определении.
        Никакой ручной регистрации не требуется.

        Координатор вызывает этот метод при build() для каждого
        зарегистрированного гейтхоста, чтобы получить список
        всех классов, которые нужно инспектировать.

        Возвращает:
            list[type] — все прямые и транзитивные подклассы
                         данного гейтхоста. Порядок: depth-first,
                         прямые подклассы перед их потомками.

        Пример:
            class RoleGateHost(BaseGateHostInspector): ...
            class BaseAction(RoleGateHost): ...
            class CreateOrderAction(BaseAction): ...
            class UpdateOrderAction(BaseAction): ...

            RoleGateHost._subclasses_recursive()
            → [BaseAction, CreateOrderAction, UpdateOrderAction]
        """
        result: list[type] = []
        for subclass in cls.__subclasses__():
            result.append(subclass)
            if hasattr(subclass, "_subclasses_recursive"):
                result.extend(subclass._subclasses_recursive())
        return result

BaseGateHostInspector определяет контракт, которому обязан следовать каждый

Инспектор — это класс, который:
1. Знает, наследников какого маркерного миксина обходить (_target_mixin).
2. Умеет инспектировать каждого наследника и собирать данные для графа.
3. Регистрируется в координаторе (GateCoordinator).

Координатор при build() обходит зарегистрированные инспекторы, вызывает
inspect() для каждого подкласса маркера и строит граф.

    @classmethod
    def _make_node_name(cls, target_cls: type, suffix: str = "") -> str:

        Формат: "module.ClassName" или "module.ClassName.suffix".
        Если модуль "__main__" или отсутствует — только "ClassName".

        Префикс типа ("action:", "role:") НЕ добавляется. Полный ключ

        Возвращает:
            str — имя узла.

            name = f"{module}.{target_cls.__qualname__}"

        Frozen dataclass требует хешируемые поля. dict не хешируем.

        на выходе иммутабельную структуру.

            **kwargs: произвольные пары ключ-значение метаданных.

        Возвращает:
            tuple[tuple[str, Any], ...] — иммутабельные метаданные.

        Пример:
            cls._make_meta(spec="admin", description="Администратор")

            cls._make_meta()
            → ()

        return tuple(kwargs.items())

    # Обход наследников
    # ═══════════════════════════════════════════════════════════════════

    @staticmethod
    def _collect_subclasses(mixin: type) -> list[type]:
        """
        Рекурсивно собирает всех наследников маркерного миксина.

        Обходит дерево наследования через Python-механизм __subclasses__(),
        который автоматически регистрирует подклассы при определении.
        Никакой ручной регистрации не требуется.

        Используется инспекторами в переопределённом _subclasses_recursive()
        для обхода наследников своего маркера (_target_mixin), а не
        наследников самого инспектора.

        Порядок обхода: depth-first. Прямые подклассы добавляются
        перед их потомками.

        Аргументы:
            mixin: маркерный миксин, наследников которого нужно собрать.
                   Например: RoleGateHost, AspectGateHost, EntityGateHost.

        Возвращает:
            list[type] — все прямые и транзитивные подклассы миксина.
                         Пустой список если у миксина нет наследников.

        Пример:
            class RoleGateHost: ...
            class BaseAction(RoleGateHost): ...
            class CreateOrderAction(BaseAction): ...

            BaseGateHostInspector._collect_subclasses(RoleGateHost)
            → [BaseAction, CreateOrderAction]
        """
        result: list[type] = []
        for sub in mixin.__subclasses__():
            result.append(sub)
            result.extend(BaseGateHostInspector._collect_subclasses(sub))
        return result

    @classmethod
    def _subclasses_recursive(cls) -> list[type]:
        """
        Рекурсивно собирает всех наследников для инспекции.

        Базовая реализация обходит наследников самого инспектора.
        Конкретные инспекторы переопределяют этот метод, чтобы
        обходить наследников _target_mixin:

            @classmethod
            def _subclasses_recursive(cls) -> list[type]:
                return cls._collect_subclasses(cls._target_mixin)

        Координатор вызывает этот метод при build() для каждого
        зарегистрированного инспектора.

        Возвращает:
            list[type] — все классы для инспекции.
        """
        result: list[type] = []
        for subclass in cls.__subclasses__():
            result.append(subclass)
            if hasattr(subclass, "_subclasses_recursive"):
                result.extend(subclass._subclasses_recursive())
        return result

- register() после build() → RuntimeError.
- Дубликат инспектора при register() → ValueError.

═══════════════════════════════════════════════════════════════════════════════
ФОРМАТ КЛЮЧЕЙ УЗЛОВ
═══════════════════════════════════════════════════════════════════════════════

Каждый узел графа идентифицируется строковым ключом "node_type:node_name".
Инспектор формирует node_type и node_name. Координатор собирает ключ:

    node_type="role", node_name="module.CreateOrderAction"
    → ключ: "role:module.CreateOrderAction"

Один класс может порождать несколько узлов от разных инспекторов.
Уникальность гарантируется комбинацией node_type + node_name.

═══════════════════════════════════════════════════════════════════════════════
ГРАФ
═══════════════════════════════════════════════════════════════════════════════

Граф построен на библиотеке rustworkx (rx.PyDiGraph). Узлы хранят dict
с полями: node_type, name, class_ref, meta. Рёбра хранят dict с полями:
edge_type, meta.

═══════════════════════════════════════════════════════════════════════════════
РАНТАЙМ-ДОСТУП
═══════════════════════════════════════════════════════════════════════════════

Координатор предоставляет типизированные методы чтения данных из графа.
Машина (ActionProductMachine) и адаптеры используют эти методы вместо
прямого доступа к ClassMetadata.

═══════════════════════════════════════════════════════════════════════════════
ПРИМЕР ИСПОЛЬЗОВАНИЯ
═══════════════════════════════════════════════════════════════════════════════

    from action_machine.metadata.gate_coordinator import GateCoordinator
    from action_machine.auth.role_gate_host_inspector import RoleGateHostInspector
    from action_machine.aspects.aspect_gate_host_inspector import AspectGateHostInspector

    coordinator = (
        GateCoordinator(strict=True)
        .register(RoleGateHostInspector)
        .register(AspectGateHostInspector)
        .build()
    )

    # Рантайм-доступ:
    spec = coordinator.get_role_spec(CreateOrderAction)
    aspects = coordinator.get_regular_aspects(CreateOrderAction)
"""

from __future__ import annotations

from typing import Any

import rustworkx as rx

from action_machine.metadata.base_gate_host_inspector import BaseGateHostInspector
from action_machine.metadata.exceptions import (
    DuplicateNodeError,
    InvalidGraphError,
    PayloadValidationError,
)
from action_machine.metadata.payload import FacetPayload


class GateCoordinator:
    """
    Центральный реестр и сборщик графа зависимостей системы ActionMachine.

    Принимает регистрацию инспекторов, строит граф через транзакционный
    build() и предоставляет типизированные методы чтения.

    Атрибуты:
        _strict : bool
            Если True — дополнительные проверки при build()
            (например, обязательность domain в @meta для Action).

        _inspectors : list[type[BaseGateHostInspector]]
            Зарегистрированные инспекторы в порядке регистрации.

        _registered : set[type[BaseGateHostInspector]]
            Множество зарегистрированных инспекторов для проверки
            дубликатов.

        _graph : rx.PyDiGraph
            Направленный граф сущностей системы. Заполняется
            при коммите (фаза 3). После build() — только чтение.

        _node_index : dict[str, int]
            Карта ключ_узла → индекс_в_графе. Заполняется при коммите.

        _class_index : dict[type, list[str]]
            Карта класс → список ключей узлов, порождённых этим классом.
            Заполняется при коммите. Используется рантайм-методами
            для поиска узлов по классу.

        _built : bool
            Флаг завершения build(). После True — register() запрещён,
            повторный build() запрещён.
    """

    def __init__(self, strict: bool = False) -> None:
        """
        Создаёт координатор с пустым графом.

        Аргументы:
            strict: если True — дополнительные проверки при build().
        """
        self._strict: bool = strict
        self._inspectors: list[type[BaseGateHostInspector]] = []
        self._registered: set[type[BaseGateHostInspector]] = set()
        self._graph: rx.PyDiGraph = rx.PyDiGraph()
        self._node_index: dict[str, int] = {}
        self._class_index: dict[type, list[str]] = {}
        self._built: bool = False

    # ═══════════════════════════════════════════════════════════════════
    # Fluent-регистрация инспекторов
    # ═══════════════════════════════════════════════════════════════════

    def register(
        self, inspector_cls: type[BaseGateHostInspector],
    ) -> GateCoordinator:
        """
        Регистрирует инспектор гейтхоста.

        Вызывается до build(). Поддерживает fluent-цепочку:

            coordinator = GateCoordinator()\\
                .register(RoleGateHostInspector)\\
                .register(AspectGateHostInspector)\\
                .build()

        Аргументы:
            inspector_cls: класс инспектора (наследник BaseGateHostInspector).

        Возвращает:
            self — для fluent-цепочки.

        Исключения:
            RuntimeError: если build() уже вызван.
            ValueError: если инспектор уже зарегистрирован.
        """
        if self._built:
            raise RuntimeError(
                f"Регистрация {inspector_cls.__name__} после build() запрещена. "
                f"Все инспекторы регистрируются до вызова build()."
            )
        if inspector_cls in self._registered:
            raise ValueError(
                f"Инспектор {inspector_cls.__name__} уже зарегистрирован."
            )
        self._inspectors.append(inspector_cls)
        self._registered.add(inspector_cls)
        return self

    # ═══════════════════════════════════════════════════════════════════
    # Построение графа
    # ═══════════════════════════════════════════════════════════════════

    def build(self) -> GateCoordinator:
        """
        Транзакционное построение графа.

        Вызывается ровно один раз после регистрации всех инспекторов.
        Три фазы: сбор → проверки → коммит. Если любая проверка
        не прошла — граф не изменяется, исключение пробрасывается.

        Возвращает:
            self — для fluent-цепочки (coordinator = GateCoordinator()...build()).

        Исключения:
            RuntimeError: если build() уже вызван.
            PayloadValidationError: невалидные поля payload (фаза 2a).
            DuplicateNodeError: конфликт ключей узлов (фаза 2b).
            InvalidGraphError: битая ссылка ребра или цикл (фаза 2c, 2d).
        """
        if self._built:
            raise RuntimeError(
                "build() уже вызван. Координатор строит граф один раз."
            )

        all_payloads, payload_sources = self._phase1_collect()
        self._phase2_check_payloads(all_payloads)
        self._phase2_check_key_uniqueness(all_payloads, payload_sources)
        self._phase2_check_referential_integrity(all_payloads)
        self._phase2_check_acyclicity(all_payloads)
        self._phase3_commit(all_payloads)

        self._built = True
        return self

    # ═══════════════════════════════════════════════════════════════════
    # Фаза 1 — Сбор
    # ═══════════════════════════════════════════════════════════════════

    def _phase1_collect(
        self,
    ) -> tuple[list[FacetPayload], dict[str, str]]:
        """
        Обходит всех инспекторов и собирает FacetPayload.

        Для каждого инспектора вызывает _subclasses_recursive(), затем
        inspect() для каждого найденного класса. payload с None
        отфильтровываются.

        Дополнительно отслеживает, какой инспектор создал каждый payload,
        для информативных сообщений об ошибках в фазе 2.

        Возвращает:
            Кортеж из двух элементов:
            - list[FacetPayload] — все собранные payload.
            - dict[str, str] — карта ключ_узла → имя_инспектора
              (для сообщений об ошибках DuplicateNodeError).
        """
        all_payloads: list[FacetPayload] = []
        payload_sources: dict[str, str] = {}

        for inspector_cls in self._inspectors:
            inspector_name = inspector_cls.__name__
            subclasses = inspector_cls._subclasses_recursive()

            for target_cls in subclasses:
                payload = inspector_cls.inspect(target_cls)
                if payload is None:
                    continue

                key = self._make_key(payload.node_type, payload.node_name)

                if key in payload_sources:
                    raise DuplicateNodeError(
                        key=key,
                        first_gate_host=payload_sources[key],
                        second_gate_host=inspector_name,
                    )

                payload_sources[key] = inspector_name
                all_payloads.append(payload)

        return all_payloads, payload_sources

    # ═══════════════════════════════════════════════════════════════════
    # Фаза 2 — Проверки
    # ═══════════════════════════════════════════════════════════════════

    def _phase2_check_payloads(
        self, payloads: list[FacetPayload],
    ) -> None:
        """
        Проверка 2a: обязательные поля payload непустые.

        Каждый payload должен иметь:
        - node_type — непустая строка.
        - node_name — непустая строка.
        - node_class — экземпляр type.

        Аргументы:
            payloads: список payload для проверки.

        Исключения:
            PayloadValidationError: если любое обязательное поле невалидно.
        """
        for p in payloads:
            if not p.node_type:
                raise PayloadValidationError(
                    node_class=p.node_class,
                    field_name="node_type",
                    detail="пустая строка",
                )
            if not p.node_name:
                raise PayloadValidationError(
                    node_class=p.node_class,
                    field_name="node_name",
                    detail="пустая строка",
                )
            if not isinstance(p.node_class, type):
                raise PayloadValidationError(
                    node_class=p.node_class,
                    field_name="node_class",
                    detail=f"ожидался type, получен {type(p.node_class).__name__}",
                )

    def _phase2_check_key_uniqueness(
        self,
        payloads: list[FacetPayload],
        payload_sources: dict[str, str],
    ) -> None:
        """
        Проверка 2b: уникальность ключей узлов.

        Дубликаты уже обнаруживаются в _phase1_collect при заполнении
        payload_sources. Этот метод — дополнительная защита на случай
        если payload_sources не используется (прямой вызов фазы 2
        в тестах).

        Аргументы:
            payloads: список payload для проверки.
            payload_sources: карта ключ → имя инспектора.

        Исключения:
            DuplicateNodeError: если обнаружен дубликат ключа.
        """
        seen: set[str] = set()
        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            if key in seen:
                raise DuplicateNodeError(
                    key=key,
                    first_gate_host=payload_sources.get(key, "unknown"),
                    second_gate_host="unknown",
                )
            seen.add(key)

    def _phase2_check_referential_integrity(
        self, payloads: list[FacetPayload],
    ) -> None:
        """
        Проверка 2c: ссылочная целостность рёбер.

        Каждое ребро ссылается на целевой узел через
        "target_node_type:target_name". Этот ключ обязан существовать
        среди собранных payload. Если цель не найдена — ребро битое.

        Аргументы:
            payloads: список payload для проверки.

        Исключения:
            InvalidGraphError: если ребро ссылается на несуществующий узел.
        """
        all_keys: set[str] = {
            self._make_key(p.node_type, p.node_name)
            for p in payloads
        }

        for p in payloads:
            source_key = self._make_key(p.node_type, p.node_name)
            for edge in p.edges:
                target_key = self._make_key(
                    edge.target_node_type, edge.target_name,
                )
                if target_key not in all_keys:
                    raise InvalidGraphError(
                        f"Ребро '{edge.edge_type}' из '{source_key}' "
                        f"ссылается на несуществующий узел '{target_key}'. "
                        f"Класс-цель не обнаружен ни одним инспектором."
                    )

    def _phase2_check_acyclicity(
        self, payloads: list[FacetPayload],
    ) -> None:
        """
        Проверка 2d: ацикличность структурных рёбер.

        Создаёт временный граф, добавляет только структурные рёбра
        (is_structural=True) и проверяет ацикличность через
        rustworkx.is_directed_acyclic_graph().

        Информационные рёбра (is_structural=False) не проверяются —
        циклические связи между сущностями допустимы.

        Аргументы:
            payloads: список payload для проверки.

        Исключения:
            InvalidGraphError: если структурные рёбра образуют цикл.
        """
        test_graph: rx.PyDiGraph = rx.PyDiGraph()
        test_index: dict[str, int] = {}

        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            idx = test_graph.add_node(key)
            test_index[key] = idx

        has_structural_edges = False
        for p in payloads:
            source_key = self._make_key(p.node_type, p.node_name)
            source_idx = test_index[source_key]
            for edge in p.edges:
                if not edge.is_structural:
                    continue
                target_key = self._make_key(
                    edge.target_node_type, edge.target_name,
                )
                target_idx = test_index[target_key]
                test_graph.add_edge(source_idx, target_idx, edge.edge_type)
                has_structural_edges = True

        if has_structural_edges and not rx.is_directed_acyclic_graph(test_graph):
            raise InvalidGraphError(
                "Структурные рёбра (depends, connection) образуют цикл. "
                "Проверьте зависимости между классами."
            )

    # ═══════════════════════════════════════════════════════════════════
    # Фаза 3 — Коммит
    # ═══════════════════════════════════════════════════════════════════

    def _phase3_commit(self, payloads: list[FacetPayload]) -> None:
        """
        Коммит всех payload в граф.

        Выполняется только если фаза 2 прошла полностью.
        Конвертирует tuple of tuples в dict для хранения в узлах графа.
        Заполняет _node_index и _class_index.

        Аргументы:
            payloads: список провалидированных payload.
        """
        # Добавляем все узлы
        for p in payloads:
            key = self._make_key(p.node_type, p.node_name)
            idx = self._graph.add_node({
                "node_type": p.node_type,
                "name": p.node_name,
                "class_ref": p.node_class,
                "meta": dict(p.node_meta),
            })
            self._node_index[key] = idx

            # Заполняем class_index
            if p.node_class not in self._class_index:
                self._class_index[p.node_class] = []
            self._class_index[p.node_class].append(key)

        # Добавляем все рёбра
        for p in payloads:
            source_key = self._make_key(p.node_type, p.node_name)
            source_idx = self._node_index[source_key]
            for edge in p.edges:
                target_key = self._make_key(
                    edge.target_node_type, edge.target_name,
                )
                target_idx = self._node_index[target_key]
                self._graph.add_edge(source_idx, target_idx, {
                    "edge_type": edge.edge_type,
                    "meta": dict(edge.edge_meta),
                })

    # ═══════════════════════════════════════════════════════════════════
    # Утилиты
    # ═══════════════════════════════════════════════════════════════════

    @staticmethod
    def _make_key(node_type: str, name: str) -> str:
        """
        Формирует уникальный ключ узла: "node_type:name".

        Аргументы:
            node_type: тип узла ("role", "action", "entity" и т.д.).
            name: имя узла ("module.ClassName").

        Возвращает:
            str — ключ вида "role:module.CreateOrderAction".
        """
        return f"{node_type}:{name}"

    # ═══════════════════════════════════════════════════════════════════
    # Публичные свойства
    # ═══════════════════════════════════════════════════════════════════

    @property
    def strict(self) -> bool:
        """Возвращает strict-режим координатора."""
        return self._strict

    @property
    def is_built(self) -> bool:
        """True если build() уже вызван."""
        return self._built

    @property
    def graph_node_count(self) -> int:
        """Количество узлов в графе."""
        return self._graph.num_nodes()

    @property
    def graph_edge_count(self) -> int:
        """Количество рёбер в графе."""
        return self._graph.num_edges()

    # ═══════════════════════════════════════════════════════════════════
    # Чтение графа — низкоуровневые методы
    # ═══════════════════════════════════════════════════════════════════

    def has_node(self, node_type: str, name: str) -> bool:
        """
        Проверяет существование узла в графе.

        Аргументы:
            node_type: тип узла.
            name: имя узла.

        Возвращает:
            True если узел существует.
        """
        return self._make_key(node_type, name) in self._node_index

    def get_node(self, node_type: str, name: str) -> dict[str, Any] | None:
        """
        Возвращает payload узла по типу и имени.

        Аргументы:
            node_type: тип узла.
            name: имя узла.

        Возвращает:
            dict с полями node_type, name, class_ref, meta.
            None если узел не найден.
        """
        key = self._make_key(node_type, name)
        idx = self._node_index.get(key)
        if idx is None:
            return None
        return dict(self._graph[idx])

    def get_nodes_by_type(self, node_type: str) -> list[dict[str, Any]]:
        """
        Возвращает все узлы указанного типа.

        Аргументы:
            node_type: тип узлов для поиска.

        Возвращает:
            Список dict с данными узлов.
        """
        return [
            dict(self._graph[idx])
            for idx in self._graph.node_indices()
            if self._graph[idx].get("node_type") == node_type
        ]

    def get_nodes_for_class(self, cls: type) -> list[dict[str, Any]]:
        """
        Возвращает все узлы графа, порождённые указанным классом.

        Один класс может порождать несколько узлов от разных инспекторов
        (например, "role:...", "action:...", "aspect:...").

        Аргументы:
            cls: класс Python.

        Возвращает:
            Список dict с данными узлов. Пустой список если класс
            не порождал узлов.
        """
        keys = self._class_index.get(cls, [])
        result: list[dict[str, Any]] = []
        for key in keys:
            idx = self._node_index.get(key)
            if idx is not None:
                result.append(dict(self._graph[idx]))
        return result

    def get_node_meta(
        self, cls: type, node_type: str,
    ) -> dict[str, Any] | None:
        """
        Возвращает meta узла указанного типа для класса.

        Ищет среди узлов, порождённых cls, узел с node_type.
        Возвращает его meta как dict.

        Аргументы:
            cls: класс Python.
            node_type: тип искомого узла.

        Возвращает:
            dict с метаданными узла. None если узел не найден.
        """
        name = BaseGateHostInspector._make_node_name(cls)
        node = self.get_node(node_type, name)
        if node is None:
            return None
        return node.get("meta")

    def get_graph(self) -> rx.PyDiGraph:
        """
        Возвращает копию графа.

        Копия — чтобы внешний код не мог модифицировать граф.

        Возвращает:
            rx.PyDiGraph — копия направленного графа.
        """
        return self._graph.copy()

    # ═══════════════════════════════════════════════════════════════════
    # Рантайм-доступ — типизированные методы
    # ═══════════════════════════════════════════════════════════════════

    def get_role_spec(self, cls: type) -> Any:
        """
        Возвращает ролевую спецификацию класса.

        Читает spec из node_meta узла типа "role" для указанного класса.

        Аргументы:
            cls: класс Action.

        Возвращает:
            str | list[str] — спецификация ролей.
            None — если класс не имеет @check_roles.
        """
        meta = self.get_node_meta(cls, "role")
        if meta is None:
            return None
        return meta.get("spec")

    # ═══════════════════════════════════════════════════════════════════
    # Строковое представление
    # ═══════════════════════════════════════════════════════════════════

    def __repr__(self) -> str:
        """Компактное строковое представление для отладки."""
        state = "built" if self._built else "not built"
        inspector_names = ", ".join(i.__name__ for i in self._inspectors)
        return (
            f"GateCoordinator("
            f"state={state}, "
            f"strict={self._strict}, "
            f"inspectors=[{inspector_names}], "
            f"nodes={self.graph_node_count}, "
            f"edges={self.graph_edge_count})"
        )
